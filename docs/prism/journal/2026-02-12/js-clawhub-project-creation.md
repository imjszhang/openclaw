## 1. 项目动机

OpenClaw 生态已有官方的 ClawHub（clawhub.ai）作为技能注册中心，但生态中缺少一个**面向用户的策展型导航站**——把散落在各处的项目、技能、教程、社区动态聚合到一个可浏览、可搜索的入口。

ClawHub 解决的是"技能怎么发布和安装"，而 JS ClawHub 要解决的是"OpenClaw 生态里有什么值得关注"。两者是互补关系，不是替代关系。

## 2. 定位与差异化

JS ClawHub 的核心定位是**策展（Curation）**，区别于自动聚合或无筛选的目录站：

|            | 一般目录站       | JS ClawHub                 |
| ---------- | ---------------- | -------------------------- |
| **筛选**   | 自动抓取或无筛选 | 人工精选，附编辑点评       |
| **内容**   | 仅外链           | 教程、指南、评测、社区动态 |
| **国际化** | 单语言           | 中英双语支持               |
| **成本**   | 不一定           | 免费、开源、纯静态         |

项目名中的 "JS" 既是作者 ID（imjszhang），也代表 JavaScript——整站纯前端实现，零后端依赖。

## 3. 技术选型

确定了纯静态站点方案，核心技术栈：

| 技术                         | 选择理由                                 |
| ---------------------------- | ---------------------------------------- |
| **纯静态 HTML**              | 零服务器成本，GitHub Pages 直接部署      |
| **Neo-Brutalism 设计**       | 大胆的视觉风格，建立品牌识别度           |
| **Tailwind CSS**             | 实用优先的响应式样式系统                 |
| **Three.js**                 | 首页 3D 动画背景，增强沉浸感             |
| **marked.js + highlight.js** | 客户端 Markdown 渲染 + 代码高亮          |
| **自研 I18nManager**         | 轻量级客户端国际化，`data-i18n` 属性驱动 |

不使用 React/Vue 等框架——对于内容导航站，原生 HTML + 模块化 JS 足够，减少构建复杂度和加载体积。

## 4. 首版功能设计

v1.0.0 一次性发布了六大模块：

### 4.1 项目导航

分类浏览 OpenClaw 生态的精选项目和集成方案。每个项目附有简介和编辑点评。

### 4.2 技能市场

展示社区创建的 OpenClaw 技能，支持详细的 Markdown 文档渲染。与 ClawHub 的区别在于更侧重"发现"而非"安装"。

### 4.3 博客系统

承载深度教程、架构分析与实践日记。支持文章系列（series）和中英双语版本（`article.md` + `article.en-US.md`）。

### 4.4 入门指南

从安装到高级配置的分步文档，面向首次接触 OpenClaw 的新用户。

### 4.5 Pulse 动态

AI 筛选的 OpenClaw 社区 X/Twitter 热点，每日自动更新。这是最有特色的模块——用 AI 从社交媒体噪音中筛选有价值的信息。

### 4.6 中英双语

完整的 i18n 实现覆盖三层：

- **UI 翻译**：所有静态文案通过 `data-i18n` 属性和语言包文件管理
- **双语 JSON 数据**：内容字段使用 `{"zh-CN": "...", "en-US": "..."}` 对象
- **双语 Markdown**：中文默认 + `.en-US.md` 英文版

导航栏一键切换语言，偏好持久化到 localStorage。

## 5. 项目架构

```
js-clawhub/
├── src/                          # 源码目录
│   ├── index.html                # 首页
│   ├── shared/
│   │   ├── js/
│   │   │   ├── i18n/             # i18n 系统
│   │   │   │   ├── index.js      # I18nManager 核心
│   │   │   │   └── locales/      # zh-CN.js, en-US.js
│   │   │   ├── nav.js            # 导航栏（含语言切换）
│   │   │   ├── footer.js         # 页脚组件
│   │   │   └── search.js         # 搜索过滤
│   │   └── css/brutal.css        # 设计系统
│   ├── blog/                     # 博客系统
│   ├── skills/                   # 技能市场
│   ├── guide/                    # 入门指南
│   └── pulse/                    # 社区动态
├── cli/                          # CLI 工具
├── build/build.js                # 构建脚本
├── docs/                         # 构建产物（GitHub Pages 部署）
└── scripts/                      # 工具脚本
```

架构设计遵循几个原则：

- **src/ 与 docs/ 分离**：源码和部署产物严格分开，构建管线负责转换
- **shared/ 复用层**：导航栏、页脚、i18n、搜索等公共模块集中管理
- **数据与展示分离**：项目、技能、博客数据通过 JSON 文件管理，HTML 模板负责渲染

## 6. CLI 工具

项目自带一个功能完整的 CLI，入口为 `cli/cli.js`，输出 JSON 到 stdout，日志输出到 stderr：

| 命令类别 | 主要命令                                                 | 用途                         |
| -------- | -------------------------------------------------------- | ---------------------------- |
| 构建部署 | `build`, `commit`, `sync`                                | 一键完成站点构建、提交和推送 |
| 数据访问 | `search`, `pulse`, `projects`, `skills`, `blog`, `stats` | 查询所有数据源               |

`sync` 命令实现了从构建到推送的全自动流程：

```bash
node cli/cli.js sync    # build → commit → push 一步完成
```

构建管线包含 7 个步骤：复制文件、注入 Google Analytics、校验 i18n 翻译完整性、清理 Pulse 数据、生成 API 层等。

## 7. 部署策略

选择 GitHub Pages 从 `docs/` 目录部署，辅以 Cloudflare DNS 配置。整个部署零成本：

- **托管**：GitHub Pages（免费）
- **CDN**：Cloudflare（免费层）
- **域名**：通过 CNAME 文件配置自定义域名
- **CI**：手动 `sync` 或命令行触发，暂未配置 GitHub Actions
