## 核心暴论

> **在不久的未来，任何无法与 AI Agent 交互的信息都是"低效信息"甚至"无效信息"！**

基于这个判断，目前有两派：

| 维度         | 本地派                               | 云端派                                   |
| ------------ | ------------------------------------ | ---------------------------------------- |
| **工具组合** | VS Code/Obsidian + Claude Code       | 云端知识库 + 云端 Agent                  |
| **优势**     | 自由度高，可随意调用喜欢的 Agent     | 信息在云端，不受设备限制，方便共享和收费 |
| **劣势**     | 内容在本地，同步有门槛，难与他人共享 | 与 Agent 互动受平台限制，自由度较低      |
| **作者立场** | 倾向于本地派，但也会尝试云端场景     | -                                        |

## 盈利模式（3 种）

### 模式 1：免费

- **适用场景**：公益性质的知识库
- **操作方式**：直接选择"免费"

### 模式 2：兑换码加入

- **适用场景**：有公域/私域流量，付费成交在外部
- **操作方式**：小红书挂虚拟商品→用户购买→发兑换码
- **优势**：适合公域引流私域成交

### 模式 3：线上支付加入

- **适用场景**：有私域流量，付费在飞书内完成
- **操作方式**：开通收款功能→设置价格→用户直接付费
- **优势**：付费直接发生在飞书上，转化路径短

**线上支付开通流程**：

1. 点击"立即开通收款能力"
2. 选择商户类型（个人/企业）
3. 填写必填内容，提交审核
4. 15 分钟后收到开通成功通知

## 自动化内容维护（OpenClaw 核心玩法）

### 案例 1：OpenClaw 中文社区知识库

**工作流**：

1. 创建飞书文档，权限设为"编辑"或"管理"（让 OpenClaw 能编辑）
2. 让 OpenClaw 抓取 GitHub Release Note → 写入飞书文档 → 生成中文总结
3. 配置 Cron Job，每 12 小时扫描一次官方 repo
4. 有版本更新自动搬运，实现无人值守维护

**扩展内容**：

- 创建作者人物小传（Peter Steinberger）
- 扒官方 guidebook 到飞书文档

### 案例 2：Dan Koe 文章知识库

- 190 篇 Dan Koe 文章自动整理
- **验证了确实有用户愿意付费**😂

## 避坑指南（3 大坑点）

### 坑 1：飞书文档写入顺序混乱

- **原因**：官方的 feishu-doc skill 不完善
- **解决方案**：需要自己打补丁

### 坑 2：每个文档都是独立工作流

- **原因**：需要依赖一套独立的 skill 组合
- **解决方案**：大量调试，稳定后封装为 Skill

### 坑 3：OpenClaw 不按期望操作

- **表现**：可能直接清空文档，或者应该清空但反而填屎山
- **解决方案**：一旦工作流稳定，立即封装为 skill 固化下来（记得测试）

**核心原则**：

> 一旦某个工作流稳定了，一定记得将这个工作流封装为一个 skill 固化下来！（记得封装好了也要测试）

## 核心结论

### 作者建议

- ❌ **不建议花钱购买别人搭建的知识库**（大概率吃灰）
- ✅ **自己动手搭建**，把能力边界往外推
- 💡 搭个知识库然后在小红书上挂虚拟商品卖钱，maybe 是个好生意

### 核心价值

> 利用 OpenClaw 等 Agent 工具自动化构建具备付费功能的云端知识库，是未来高价值信息变现的有效路径。

用户应亲自掌握搭建与调试技能，将不稳定的工作流固化为 Skill，从而打造可持续盈利的自动化内容资产。
