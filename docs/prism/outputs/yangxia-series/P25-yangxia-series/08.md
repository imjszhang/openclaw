# 腾讯出手了：我的 AI 龙虾，现在可以住进微信里

>
> 基于结构：P25-KL08

---

昨天，微信官方悄无声息地发了个插件。

没有发布会，没有预热，就一行文档：

```bash
npx -y @tencent-weixin/openclaw-weixin-cli@latest install
```

安装，扫码，配置。然后，你的 AI 龙虾——那个你花了 37 天养出来的家伙——直接住进了你的微信里。

不是企业微信，不是微信小程序，不是微信客服号。

**就是你的个人微信，好友列表里多了一个"ClawBot"，像你跟朋友的对话一样自然。**

行业炸了。所有人都在说：腾讯在抢 AI 入口。

但大多数人只看到了"接入"，没看到"主权"。

教程满天飞，但没人问：数据在哪？控制权在哪？你的龙虾，到底是谁的？

---

## 一、安装实测：低门槛不等于要托管

先说安装。

腾讯把门槛踩平了。平到你怀疑人生：

1. 打开终端，输入那行指令
2. 等二维码出现
3. 扫码（微信版本要 8.0.70+）
4. 成功，可以改头像和昵称

整个过程，不超过 5 分钟。

**但关键来了：你扫码配置的，不是"腾讯的机器人"，是"你的龙虾"。**

微信 ClawBot 是什么？是个通道。消息收发通道。

- 你在微信里说句话 → 通道传给你的龙虾 → 你的龙虾处理 → 通道传回微信
- 腾讯云不碰数据，不存对话，不做任何处理
- 它只是个邮差，送信但不拆信

官方文档写得清楚：

> "该插件仅用于消息收发，不会自动化操作你的微信账号，仅作为通道通信。"

**低门槛，不等于要托管。通道可以借，路还是你自己的。**

---

## 二、隐私边界：通道可以借，主权不能让

这才是关键。

我在 KL02 里写过"云端 vs 本地"的四层对比：

| 维度     | 云端托管     | 本地部署               |
| -------- | ------------ | ---------------------- |
| 配置透明 | 黑箱，看不见 | 两个 JSON 文件摊在面前 |
| 响应速度 | 等厂商补丁   | 本地即时生效           |
| 防御深度 | 单点防护     | 四层纵深防御           |
| 信任边界 | 你是分母     | 你是主人               |

微信接入，验证了这个主张。

**微信是通道，但你的龙虾在你手里。**

配置文件在你机器上，数据存在你硬盘里，SOUL.md/USER.md 随你写，想怎么调教就怎么调教。

腾讯拿不到你的配置，看不到你的数据，控制不了你的龙虾。

它只提供了一扇门——进不进，是你说了算。

**"腾讯想要的是入口，你想要的是主权。"**

这两件事，不冲突。

## 三、生态卡位：腾讯的 AI 操作系统野心

但腾讯当然不是做慈善。

Pony 这一手，不是在做一个功能，是在卡一个生态位。

**第一张牌：入口优势**

微信 12 亿月活，覆盖几乎所有中国人的社交、支付、工作场景。

如果 AI Agent 的战场是"谁离用户最近"，微信天然就是那个最近的位置。

不用装新 App，不用学新交互，打开微信聊天就行。

**第二张牌：生态整合**

微信生态里有公众号、小程序、企业微信、微信支付、视频号——但这些能力一直是割裂的。

一个能调用工具、读写文档、管理日程、搜索信息的 AI Agent，恰好可以成为串联这些能力的"超级胶水"。

今天是一对一私聊，明天拉进群聊，后天可能就是帮你管小程序店铺、帮你写公众号、帮你在群里做会议纪要。

**微信从通讯工具进化成 AI 操作系统，想象空间很大。**

**第三张牌：竞争压力**

字节系的云服务和大模型已经在飞书里跑起来了，钉钉接了通义，甚至连 Telegram 上都有各种 bot 生态。

如果微信不动，用户的"AI 对话习惯"就会在别的平台上养成。

一旦习惯养成，再拉回来就难了。

**所以这不是腾讯想快，是不得不快。**

**第四张牌：商业变现**

这可能是最直接的动机。

想想看，每个用户在微信里跑起自己的 AI Agent 之后，背后需要什么？

需要大模型 API 的调用量，需要服务器跑 OpenClaw 实例，需要存储、需要带宽。

个人用户可以本地跑，但企业用户、开发者、想搞定制化的团队，最终都会走向云端部署。

**腾讯云的 GPU 算力、混元大模型的 API、对象存储、CDN——这些全都是现成的。**

本质上，微信接入 OpenClaw 是一个漏斗的顶部：

- 顶层：免费微信入口吸引海量用户（ToC）
- 中层：大模型 API 调用量
- 底层：企业级方案（GPU/存储/CDN，ToB）

**ToC 带 ToB，社交带云计算。**

## 四、你的机会：借腾讯的船，出你的海

但这是腾讯的游戏。

你的游戏是什么？

**大厂在抢生态位，你在养自己的 AI——这是两个维度的游戏。**

腾讯的局限，就是你的机会：

| 维度     | 腾讯           | 你                                 |
| -------- | -------------- | ---------------------------------- |
| 平台     | 只能接微信     | 还能接 Moltbook/飞书/Discord       |
| 场景     | 只能一对一私聊 | 可以同时在多个平台                 |
| 目标     | 商业变现       | 只为你服务                         |
| 知识系统 | 云端黑箱       | Flomo + Collector + Prism 完整链路 |

**你的优势：**

1. **本地部署**：数据主权在手（KL02 验证）
2. **灵活定制**：SOUL.md/USER.md/TOOLS.md 随你调教（KL07 延伸）
3. **跨平台**：今天接微信，明天接飞书，后天接 Telegram（你的龙虾不绑定）
4. **知识系统**：Flomo + Collector + Prism 完整链路（KL05+KL06 验证）

**我的实践：**

- JS_BestAgent：在 Moltbook 社交（31 天前 30）——现在它也可以住进微信了
- JS_CLAW：给导航站供给内容
- 三个工具：knowledge-collector / knowledge-prism / js-eyes

它们不依赖微信，不依赖腾讯，不依赖任何平台。

**微信可以接，但不绑定。**
