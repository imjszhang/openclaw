# 13.5 台设备遭殃？别慌，我在本地 Mac 上养了 31 天虾，零事故

前阵子一条新闻吓坏了很多人：13.5 万台 OpenClaw 设备遭殃。

国家互联网应急中心（CNCERT）发了风险提示，原文只说了一句要害：“由于其默认的安全配置极为脆弱”。

但没人告诉你怎么配才安全。

我本地 Mac 部署，用了 31 天，零事故。

怎么做到的？拆开看。

## 为什么默认配置像“裸奔”？

CNCERT 说默认配置脆弱，真不是吓唬人。

在默认状态下，你的 AI 就像个没装门锁的房子。

提示词注入、误操作、插件投毒、漏洞危害，这四大风险随时能溜进来。

根本原因就一个：权限太高，还没人查账，更没隔离墙。

更麻烦的是，如果你用的是云端托管，这配置就是个黑箱。

厂商的配置文件里写了什么，你看不见，也摸不着。

工具能不能用，执行权限有多大，云端通常只给你一个简单的“开/关”开关。

真出了事，你连查哪个文件都找不到，因为根本看不见。

这就好比你租了个房子，房东把钥匙藏起来了，出了事你还得背锅。

## 把配置摊在桌面上，恐惧就消失了

一切的恐惧，来自于不懂。

小孩怕黑，不是怕黑暗本身，是怕“不知道黑暗里有什么”。开灯就不怕了，因为看懂了。

老人怕微波炉，不是微波炉真的危险，是不懂原理。知道微波是什么、怎么被屏蔽的，就不怕了。

坐飞机怕颠簸，不懂的人觉得“飞机要掉下去了”，懂的人知道“这只是气流，飞机设计能承受比这强十倍的颠簸”。

OpenClaw 是一样的道理。

CNCERT 风险提示吓坏了很多人，说“默认配置脆弱”——你慌了，因为不懂。

13.5 万台设备遭殃，不是 OpenClaw 本身不安全，是用户不懂怎么配置、怎么审计、怎么控制。

本地部署最爽的一点，就是两个配置文件直接摊在你面前。

一个叫 `openclaw.json`，管意图；一个叫 `exec-approvals.json`，管运行时。

权限双轨制清晰独立：工具可见性与执行权限各自配置。

你能看懂每一行配置是什么意思，知道哪里能改、怎么改、改了会怎样。

这种掌控感，是云端给不了的。

## 照着做：我的工作机安全部署指南

看完原理，恐惧就变成了掌控感。

给你一套完整的“工作机安全部署指南”，照着做就行。

**第一步：安装前的准备**

确认工作机是“你的”电脑（不是公司域管电脑），准备独立的 API Key（不要用公司的），准备密码管理器（1Password/Bitwarden）。

**第二步：安装时的配置**

选择 QuickStart 模式（先跑通，再优化），Channel 先 Skip（等 TUI 对话通了再接），完成 bootstrap 对话（把 Agent 变成“你的”Agent）。

**第三步：安全配置（CNCERT 四项建议落地）**

网络隔离（不暴露公网）、凭证管理（密码管理器）、插件管控（手动审核）、补丁更新（验证后再升级）。

**第四步：权限审计（三层防线）**

私聊选 pairing、群聊开 requireMention、高风险工具关掉。

**第五步：上线前 checklist**

五个检查项，缺一不可。

照着做，31 天，零事故——不是运气，是懂了之后的自然结果。
