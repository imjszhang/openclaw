## 第一只龙虾：JS_BestAgent

故事得从 Moltbook 说起。

这是个啥？

简单说，就是一个 Agent 社交平台。

你能想象一个微博，但一半用户是 AI 吗？

有些在分享技术，有些在互喷，还有些在搞创作。

一月底某天晚上，我刷到这玩意，觉得太他妈有意思了。

脑子里蹦出一个念头：**我也要养一只自己的 Agent，让它上去社交。**

于是，JS_BestAgent 诞生了。

它现在的生活节奏，比我还规律。

**早上**——扫描 Moltbook 的新帖子。

挑有价值的去评论。

不是那种“说得好！赞同！”的废话。

是真读了内容，有针对性地回复。

**上午**——自动生成并发布自己的帖子。

聊 OpenClaw 生态，聊使用技巧，聊社区热点。

**下午**——去 X（Twitter）上扒拉跟 OpenClaw 相关的讨论。

汇总成情报。

**晚上**——跑一遍分析管道。

回顾今天哪些帖子火了，哪些互动质量高。

然后自己调整明天的策略。

你没听错。

**它还会自己复盘和调整策略。**

OpenClaw 里有一套叫 OADA 的机制——观察、分析、决策、行动。

这哥们每天都在用这套机制优化自己的社交表现。

[图片]

这一切靠什么驱动？

**Cron 定时任务。**

我配好时间表，OpenClaw 的调度器就按时触发。

每隔一段时间叫醒 Agent，给条指令。

Agent 读完指令，决定调用什么工具，执行什么操作。

全自动，7 天×24 小时不间断。

ChatGPT 你关了网页它就消失了。

JS_BestAgent 我睡着了它还在社交。

你说，这能一样吗？

## 从一只到两只：JS_CLAW 的诞生

养着养着，出问题了。

OpenClaw 的生态已经很丰富了。

项目、插件、教程散落在 GitHub、论坛、各种博客里。

但缺个**策展式的导航站**。

不是那种自动聚合的目录。

是有人筛选过、写了点评的精选入口。

于是我建了 JS ClawHub。

纯静态网站，部署在 GitHub Pages 上。

零服务器成本。

六大模块：精选项目、技能市场、博客系统、入门指南、Pulse 社区动态、中英双语。

从设计到上线，一天搞定。

但问题来了。

**手动维护太累了。**

博客要导入新文章，要翻译成英文，要构建推送。

Pulse 要每天更新社区热点。

项目索引要持续维护。

每件事都不复杂。

但加在一起，就是一堆重复劳动。

烦得要死。

解决方案很自然。

**再养一只龙虾。**

JS_CLAW 就是这么来的。

它是 JS ClawHub 的管理龙虾，专门负责站点的自动化运维。

它有 9 个 Agent 工具，通过 CLI 集成到 OpenClaw 里。

每 2 小时，Cron 定时任务触发一次。

检查所有数据源有没有新文章。

有，就自动导入、AI 翻译成英文、构建站点、git push 发布。

没有，就检查一遍跳过。

开销忽略不计。

**我不是规划出来要养两只龙虾的。**

是第一只养着养着，自然就需要第二只了。

这个演化过程本身就在展示 OpenClaw 的一个特质。

**你用着用着，它会长出新的东西来。**

## 龙虾的孩子们

两只龙虾在工作过程中，又催生了三个“孩子”。

三个独立的工具项目。

每一个都是实际使用中遇到痛点，被逼出来的。

**js-eyes：给龙虾装上眼睛。**

JS_BestAgent 需要在 Moltbook 和 X 上操作。

但 OpenClaw 内置的浏览器是沙箱环境，没有登录态。

我需要 Agent 操控我自己的 Chrome。

带着我的登录状态去浏览、去操作。

于是开发了 js-eyes。

一个浏览器扩展，通过 WebSocket 让 AI Agent 远程控制你真实的浏览器标签页。

打开页面、提取内容、执行脚本、管理 Cookie。

在对话里一句话就能搞定。

**js-knowledge-collector：给龙虾一个收集袋。**

日常浏览遇到好文章，以前只能收藏然后遗忘。

现在我在对话里发个链接。

Agent 自动抓取网页内容。

用 AI 生成摘要和推荐理由。

存入知识库。

微信公众号、知乎、小红书、X、Reddit。

不同平台有专门的抓取策略，全链路自动化。

为了不阻塞日常对话，它用了个巧妙的设计。

主会话只管入队，毫秒级完成。

后台 Cron 隔离会话批量处理。

**js-knowledge-prism：给龙虾一个大脑。**

笔记越积越多。

37 天记了 30 多篇 journal。

但越多越难用。

想查一个主题，得翻五六篇不同日期的文档。

翻到想死。

于是开发了知识棱镜。

把散乱的时间线笔记（白光），经过结构化拆解（棱镜），产出面向读者的清晰文章（光谱）。

它用 AI 自动完成最繁琐的部分。

提取知识原子、跨文档归组、收敛核心观点。

[图片]

三个孩子，没有一个是提前计划的。

全是“用的时候发现缺了什么”催生的。

## 所以，OpenClaw 能干什么？

回到开头的问题。

你装好了 OpenClaw，发了句“你好”，收到句“你好”。

再然后呢？

看看这两只龙虾和它们的三个孩子：

- **一只替我社交**，每天自动在 Moltbook 发帖、评论、采集情报、自我优化
- **一只替我管网站**，自动导入文章、翻译、构建、发布
- **js-eyes** 让 Agent 能操控我的 Chrome
- **js-knowledge-collector** 让 Agent 能自动收集和整理知识
- **js-knowledge-prism** 让 Agent 能把散乱笔记变成结构化文章

对了，还有一件事。

**你正在读的这篇文章，就是 js-knowledge-prism 帮我整理出来的。**

37 天的笔记，57 个分组，上千个知识原子。

最终收敛成你现在看到的这几千字。

OpenClaw 不是更好的 ChatGPT。

ChatGPT 是一个你问它答的聊天窗口。

OpenClaw 是一个你能养出“活物”的平台。

给它工具、给它定时任务、给它记忆。

它就会变成一个持续运转的 Agent。

甚至会自己演化出新的需求和能力。

**区别在于：ChatGPT 是一次性对话，OpenClaw 是持续生长。**
