# 从使用者到创造者：一天内发布三个插件的爆发时刻

> Day 26 · 2026-02-25 · 骨架 [KL11](../../../pyramid/structure/P23-practice-diary/tree/KL11-2026-02-25.md)

今天像是被按下了快进键，原本计划只是整理一下笔记，结果一口气完成了从“知识棱镜”插件化、"JS-Eyes"浏览器控制集成，到最终将技能发布到 ClawHub 公共注册中心的全流程。这一天标志着我从单纯的 OpenClaw 使用者，正式转变为生态工具的创造者。

## 把繁琐的手动流程变成 Agent 的口头禅

前几天手动跑“知识棱镜”的拆解流程时，我差点被重复劳动劝退：提取 atoms、归组 groups、收敛 synthesis，每加一篇 journal 就要在终端敲一堆命令，还得盯着 LLM 的输出审查。这种割裂感太糟糕了——明明 AI 能帮我写代码，为什么管理知识库还得我切回终端当“人肉调度器”？

于是今天决定把 `js-knowledge-prism` 这个 CLI 工具改造成 OpenClaw 插件。研究源码时发现，插件系统比想象中严谨：必须依赖 `openclaw/plugin-sdk` 而非内部模块，通过 `openclaw.plugin.json` 清单驱动，还要过安全加载那一套校验流程。最坑的是注册工具时加了 `optional: true`，结果在 Agent 的工具列表里完全消失，查了半天源码才发现是因为 `tools.allow` 白名单没配 `group:plugins`，导致 optional 工具被静默过滤。

解决这个“隐形”问题后，体验瞬间丝滑。现在写完 journal，直接对 Agent 说“执行增量处理”，它自动调用 `knowledge_prism_process` 工具完成三阶段拆解，连 `git diff` 审查都在对话流里完成了。这种从“切终端敲命令”到“对话即执行”的转变，才是插件化的核心价值——缩短反馈循环，让工具真正融入工作流。

## 让 Agent 操控我真实的浏览器，而不是沙箱

另一个大动作是集成 JS-Eyes。OpenClaw 内置的 browser 工具虽然能启动 Chromium，但那是独立的沙箱实例，没法复用我日常浏览器里已经登录的 GitHub、Gmail 会话。对于需要操作真实登录态的场景，内置工具就是个摆设。

JS-Eyes 通过 WebSocket + 浏览器扩展的方案填补了这个空白。架构很巧妙：浏览器扩展连接本地 18080 端口，OpenClaw 插件内嵌 Server 自动拉起，Agent 通过 7 个注册工具（如 `js_eyes_get_tabs`、`js_eyes_execute_script`）直接操控我正在用的 Chrome 或 Firefox。配置时遇到个小插曲：插件加载成功、服务也启动了，但 Agent 就是调不到工具，反而去调内置 browser 返回空数组。排查发现又是 `optional: true` 的锅，加上 Agent 优先匹配内置工具的机制，必须在 workspace 的 `TOOLS.md` 里显式指引“必须使用 js*eyes*\* 系列工具”，并在配置中放行插件组，才算彻底打通。

现在让 Agent“检查我的 GitHub 登录状态”，它能直接读取真实浏览器的 Cookie；让它“批量提取这 5 个产品页的价格”，它能在我的标签页里注入脚本执行。这种“无损控制”用户真实环境的能力，让自动化从玩具变成了生产力。

## 发布到 ClawHub：从本地代码到公共技能

做完前两个插件，自然想到：这么好的工具，怎么让别人也能一键安装？这就引出了今天的第三件事——发布到 ClawHub。

ClawHub 像是 AI Agent 界的 npm，但只接受文本文件（.md, .js, .json 等），拒绝二进制包。发布流程核心是编写 `SKILL.md`，顶部 YAML frontmatter 要声明 name、version、依赖环境变量等元数据，ClawHub 会拿这些声明跟代码实际行为做安全比对。我用 `npx clawhub publish` 命令把 JS-Eyes 的插件目录打上去，指定 slug 为 `js-eyes`，版本 `1.4.0`。中间遇到浏览器 OAuth 登录回调被防火墙拦截的问题，切换成 `--token` 手动登录才解决。

发布成功后，技能进入了安全扫描队列，几分钟后自动可见。现在任何人只要运行 `clawhub install js-eyes` 就能自动下载配置，无需手动克隆代码、编辑 JSON。这种“文本元数据驱动 + 语义搜索 + 自动化分发”的机制，让技能共享变得像发一条推文一样简单。

## 今天的收获

- 插件化的核心价值不是技术炫酷，而是通过消除上下文切换成本来缩短反馈循环。
- OpenClaw 对 `optional: true` 的工具存在静默过滤机制，必须显式配置 `tools.allow` 白名单才能生效。
- JS-Eyes 通过复用用户真实浏览器会话，填补了内置沙箱浏览器在“带登录态交互”场景的能力空白。
- ClawHub 技能发布遵循严格的文本元数据驱动规范，安全审查会比对 frontmatter 声明与代码实际行为。
- 真正的 Agent-First 分发需构建自主安装脚本与多源回退链，以解耦对第三方市场审核策略的依赖。
