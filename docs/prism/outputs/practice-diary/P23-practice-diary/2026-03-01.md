# 核心层零改动的胜利：Agent-First 五层架构落地实录

> Day 30 · 2026-03-01 · 骨架 [KL22](../../../pyramid/structure/P23-practice-diary/tree/KL22-2026-03-01.md)

昨天还在纠结怎么把散乱的笔记变成可复用的知识，今天直接给整个项目动了一次“大手术”——从单纯的 CLI 工具进化为完整的 Agent-First 五层架构。这不仅是代码重组，更是思维模式的切换：以前想着怎么让人好用，现在得先让 AI Agent 能“读懂”并自动调用。

## 给 AI Agent 造个“家”：五层架构的实战升级

起因是复盘了 JS-Eyes 项目的代码，突然意识到那种以 AI 为第一消费者的设计模式太值得系统化了。于是今天决定把 `js-knowledge-prism` 从原本的"CLI + Plugin"结构，强行拉升到完整的五层架构：Agent 接口层、业务核心层、人类 CLI 层、开发工具链层、扩展技能层。

最让我兴奋的实施结果是：**核心业务逻辑层（Layer 2）完全没动**。

以前总担心重构会牵一发而动全身，但这次验证了“核心逻辑写一次”的理念。所有的构建、测试、分发、扩展能力，都是像插件一样围绕核心层“长”出来的。Layer 1 只是薄薄的一层包装，把核心函数映射成 Agent 能调用的 Tool；Layer 3 留给人类用户命令行交互；而新增的 Layer 4 和 Layer 5 则彻底解放了生产力。

特别是在 Layer 4 开发工具链上，我新建了 `cli/cli.cjs`，一口气实现了 `build`、`bump`、`commit`、`sync`、`release` 五个命令。以前发版要手动改三个地方的版本号（package.json, plugin.json, SKILL.md），现在一条 `bump` 命令全自动同步。构建系统也更智能了，能同时输出主技能包、子技能包和 `skills.json` 注册表，真正做到了“提交即发布”。

## 踩坑与填坑：CJS 与 ESM 的模块战争

当然，过程也不是一帆风顺。在开发 Layer 4 工具链时，我遇到了一个典型的 Node.js 模块系统“坑”。

项目根目录的 `package.json` 里设了 `"type": "module"`，这意味着所有 `.js` 文件默认都被当成 ESM 处理。当我兴冲冲地在新建的开发 CLI 里写下 `require('archiver')` 时，控制台直接报错：`require is not defined`。

起初我想把整个项目改成 CommonJS，但转念一想，核心层和 Plugin 层大量使用了 `import/export` 语法，改动成本太高。最后的解决方案很“极客”：把开发 CLI 的文件扩展名统一改为 `.cjs`。

```bash
# 最终的文件结构
cli/
├── cli.cjs          ← 入口文件，强制使用 CJS
└── lib/
    ├── builder.cjs  ← 多目标构建逻辑
    └── git.cjs      ← Git 操作封装
```

这个小决策让我深刻体会到：在混合架构中，文件扩展名不仅是后缀，更是隔离不同模块系统的防火墙。用 `.cjs` 明确告诉 Node.js“这里用老规矩”，既避免了与项目其余 ESM 模块的冲突，又让我能直接使用 `require()` 引入那些还没完全适配 ESM 的老牌库（比如 `archiver`）。

## 让技能“自己长出来”：扩展生态的闭环

架构升级的最后一块拼图是 Layer 5 扩展技能层。以前加新功能得改主代码，现在只需要在 `skills/` 目录下扔一个新文件夹。

今天我创建了第一个子技能 `prism-output-blog`，它能自动把视角笔记转化成博客文章。神奇的是，主插件里的 `knowledge_prism_discover_skills` 和 `knowledge_prism_install_skill` 这两个新 Tool，能让 Agent 自己发现并安装这个技能。

构建系统扫描到 `skills/*/SKILL.md` 里的 YAML frontmatter，自动生成 `skills.json` 注册表。Agent 只要 fetch 这个静态 JSON，就能知道有哪些新技能可用，甚至能自动下载、解压、安装依赖。这种“静态注册表 + 动态发现”的机制，让技能生态真的有了自生长的能力。

## 今天的收获

- 架构分层的核心价值在于“核心逻辑写一次”，让 Agent、CLI、测试都能复用同一套业务底座。
- 混合模块系统中，利用 `.cjs` 扩展名是隔离 ESM 与 CommonJS 冲突的轻量级银弹。
- 静态技能注册表（skills.json）比动态 API 更适合 Agent 自动发现，因为它无服务器依赖且构建时一致。
- 开发工具链的自动化（build/bump/sync）是将开发者从重复劳动中解放出来的关键基础设施。
- 知识体系的生命力不在于一次性完美设计，而在于每次新输入都能触发增量蒸馏的流水线。
