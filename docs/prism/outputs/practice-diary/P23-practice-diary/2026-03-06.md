# 把独立工具塞进 Agent 身体里：知识收集器插件化实战

> Day 35 · 2026-03-06 · 骨架 [KL17](../../../pyramid/structure/P23-practice-diary/tree/KL17-2026-03-06.md)

今天要把刚成型的 `js-knowledge-collector` 从独立 CLI 工具改造为 OpenClaw 插件，让 AI Agent 能直接调用抓取和总结能力，同时把原本跑在 3000 端口的 Web UI 无缝挂载到 Gateway 上。

## 纯 ESM 项目的“水土不服”与动态导入

参照 `JS-Eyes` 的模板，我很快搭好了 `openclaw-plugin/` 目录结构，但在 `index.mjs` 入口文件中踩了个典型的坑。习惯性地照搬了旧写法，用 `createRequire(import.meta.url)` 去 `require` 项目内部的 `collector.js` 模块，结果直接抛出 `ERR_REQUIRE_ESM`。

这才反应过来，我的项目本身已经是 `"type": "module"` 的纯 ESM 环境，所有 `.js` 文件都是 ES 模块，CommonJS 的 `require` 根本加载不了。解决思路很简单：去掉 `createRequire` 的包装，直接使用动态 `import()`。

```javascript
// 错误写法
const { collect } = require("../cli/lib/collector.js");

// 正确写法
const { collect } = await import("../cli/lib/collector.js");
```

这个改动不仅解决了报错，还意外带来了性能红利：动态导入是懒加载的，只有当 Agent 真正调用 `knowledge_collect` 工具时，对应的业务逻辑才会被加载，减少了插件启动时的内存占用。

## 让 Web UI“寄生”在 Gateway 路由下

原本项目有一个独立的 HTTP 服务器跑在 `localhost:3000`，现在的目标是让它通过 OpenClaw Gateway 的同一端口（默认 18789）对外服务。查阅文档发现，旧版的 `api.registerHttpHandler()` 已经在 2026.3.2 版本中被移除，必须改用 `api.registerHttpRoute()`。

最让我惊喜的是前端兼容的处理。原前端 `src/index.html` 中的 API 请求写的是相对路径（如 `fetch('api/v1/articles.json')`），没有前导斜杠。这意味着当页面通过 `http://localhost:18789/plugins/knowledge/` 访问时，浏览器会自动将相对路径解析为 `.../plugins/knowledge/api/v1/articles.json`，正好命中我在插件中注册的路由。

**完全不需要修改前端代码**，这种“无心插柳”的兼容省去了大量重构工作。

不过在处理静态文件通配路由 `/plugins/knowledge/{filePath}` 时，我特意加了双重保险：一方面依赖 OpenClaw 路由系统“具体路径优先”的原则，另一方面在 handler 内部显式检查，如果子路径以 `api/` 开头直接返回 404，防止通配符意外拦截了 API 请求。此外，针对 SQLite 数据库连接，我也从“长连接”改为了“按请求开关连接”，避免在 Gateway 长期运行中持有文件句柄导致锁定问题。

## 技能与数据的“分家”

为了让 `link-collector` 技能随插件自动部署，我在 `openclaw.plugin.json` 中声明了 `"skills": ["./skills"]`。这里做了一个关键的设计决策：**技能文件与运行时数据分离**。

技能定义文件（`SKILL.md` 等）随插件走，作为只读资源被所有 workspace 共享；而运行时产生的队列文件（`inbox.jsonl`、`batch-*.jsonl`）则必须留在各个 workspace 的 `.openclaw/link-collector/` 目录下独立存储。这样既保证了技能逻辑的统一更新，又确保了不同 workspace 的数据隔离和并发安全。

## 今天的收获

- 纯 ESM 项目内部模块引用必须使用动态 `import()`，`createRequire` 会导致 `ERR_REQUIRE_ESM`。
- 前端使用无前导斜杠的相对路径 API 请求，可天然适配 Gateway 的插件路由前缀。
- 在 Gateway 路由中操作 SQLite 应采用“按请求开关连接”模式，避免长期持有文件句柄。
- 插件技能文件应只读共享，运行时数据必须隔离在 workspace 本地以保证并发安全。
- 耗时任务（如 LLM 总结）必须通过 inbox/batch 轮转机制剥离出主会话，防止阻塞交互。
