# 给知识图谱装上"Z 轴”：Prism 记忆桥接与 3D 可视化实战

> Day 39 · 2026-03-10 · 骨架 [KL24](../../../pyramid/structure/P23-practice-diary/tree/KL24-2026-03-10.md)

昨天把数据清洗的管道铺好后，今天终于到了“验收成果”的时刻：一方面要让 Prism 真正连通 OpenClaw 的记忆核心，另一方面得把那个扁平的 2D 图谱升级成能旋转、能溯源的 3D 金字塔。原本以为只是改几个配置和换个库，结果在硬件门槛和渲染细节上还是踩了几个坑。

## 记忆桥接的“硬门槛”：没有 Embedding Provider 一切免谈

上午的重头戏是打通 Prism 到 OpenClaw 的记忆桥接。我的设想很美好：Prism 负责从金字塔结构中筛选出信息密度最高的 Groups 层（忽略噪声大的 Journal 和 Outputs），利用 `mtime` 增量同步机制，把清洗后的高价值知识推送到 OpenClaw 的 `extraPaths` 中。

但在配置 `memorySearch.extraPaths` 时，我遭遇了“静默失败”——新导入的 Markdown 文件死活搜不到。排查半天才发现一个容易被忽视的硬性约束：**向量检索的开启不仅依赖 `memorySearch.enabled = true`，还必须配置至少一个可用的 Embedding Provider。**

如果没有 Provider（无论是远程的 OpenAI 还是本地的 Ollama），OpenClaw 的 `syncMemoryFiles` 和 `indexFile` 函数会直接返回，连全文索引（FTS）都不会建立。这意味着，哪怕你配置了 `extraPaths`，新内容也永远进不去检索库。

解决过程倒是意外地顺畅。考虑到隐私和低成本，我放弃了调用远程 API，转而在本地部署轻量模型。对比了 Ollama 和 `node-llama-cpp` 后，我选择了后者，因为它不需要常驻后台进程。只需在 `package.json` 中运行 `pnpm approve-builds` 勾选模块，再执行 `pnpm rebuild`，它就能自动下载约 0.6GB 的 `EmbeddingGemma-300M` 模型。配置生效后，Prism 导出的 73 个高价值文件在 0.6 秒内完成了索引，检索质量明显优于直接投喂原始笔记。

## 3D 图谱升级：从“平面图”到“可旋转的金字塔”

下午的时间全花在了知识图谱的可视化升级上。之前的 2D D3 方案在节点超过 30 个时就显得拥挤不堪，且无法直观展示“期刊→观点→输出”的六层金字塔结构。

这次我决定引入 `3d-force-graph`（基于 Three.js 封装）。最大的挑战在于布局算法：如何在三维空间中自然地呈现层级？我尝试了多种力导向配置，最终发现利用 `d3.forceY` 将不同层级的节点强制分布到不同的 Y 轴高度是最高效的。当用户在浏览器中左键旋转视图时，一个清晰的金字塔结构赫然眼前，Journal 在底层，Outputs 在塔尖，中间是凝练的 Groups。

为了让体验更极致，我还做了两个关键优化：

1.  **离线内联**：为了避免 CDN 加载失败或网络延迟，我写了一段正则逻辑，自动检测 `vendor/` 目录下的 `three.min.js` 等四个核心库，并将它们内联到生成的 HTML 中。这让图谱文件可以完全离线运行。
2.  **全链路溯源高亮**：这是今天最“爽”的功能。点击任意节点，系统会通过 BFS 双向遍历算法，瞬间高亮其上游所有祖先（直到 Journal）和下游所有子孙（直到 Output），其余节点自动暗化。配合相机自动飞向目标的动画，那种“顺藤摸瓜”的掌控感是 2D 图表无法比拟的。

测试时，渲染 33 个节点和 41 条关系流畅度满分，之前的卡顿感一扫而空。

## 今天的收获

- **记忆桥接的前提**：仅开启 `memorySearch` 开关无效，必须配置至少一个 Embedding Provider（本地或远程）才能触发新内容的索引建立。
- **低成本本地方案**：`node-llama-cpp` 配合 300M 参数模型是低配硬件上实现私有化向量检索的性价比之选，无需常驻后台。
- **数据筛选的价值**：Prism 通过只同步 Groups 层并忽略噪声数据，能将向量检索的数据准备成本降低 80% 以上，同时提升检索信噪比。
- **3D 布局的核心**：利用 `d3.forceY` 强制垂直分层，是解决知识图谱在三维空间中结构混乱、无法直观展示金字塔层级的关键。
- **交互即洞察**：BFS 双向遍历实现的全链路溯源高亮，让用户能瞬间理解单个知识点在整个知识体系中的上下游位置。
