# 让日记自己长出来：从手动触发到无人值守的自动化跨越

> Day 40 · 2026-03-11 · 骨架 [KL25](../../../pyramid/structure/P23-practice-diary/tree/KL25-2026-03-11.md)

昨天还在为 3D 图谱的升级兴奋，今天就得面对更现实的痛点：我的知识处理链路还是“半自动”的。Journal 变 Synthesis 能自动跑，但 Synthesis 变 Structure、Structure 变 Output 还得我手动敲命令或求 AI 代劳。今天的目标很明确：把后两段链路也打通，实现真正的“写完即发布”无人值守闭环。

## 把三个 Cron 砍成两个：解决时序同步的“贪多嚼不烂”

一开始我的直觉是“各司其职”：既然链路有三段（Journal→Synthesis→Structure→Output），那就配三个 Cron 任务，每段独立运行。但跟 Cursor Agent 推演了一下，立刻发现了坑：如果 Structure 刷新和 Output 生成拆成两个独立定时任务，一旦前者还没跑完后者就启动了，或者两者时序错乱，生成的文章要么是空的，要么是旧数据。为了解决这个时序同步，我得引入复杂的锁机制或状态检查，配置负担重且容易出错。

果断放弃“独立系统”方案，采用**整合策略**。我将后两段合并为一个 `prism-auto-output` 任务（默认 120 分钟运行一次），内部划分为两个阶段：Phase 1 专门负责检测 Synthesis 和 Groups 的变化，有变动才刷新 Structure（SCQA + Key Lines）；Phase 2 紧接着检测 Structure 目录的 mtime，有变动才生成 Output。

这种“串联”设计不仅消除了时序竞争，还天然实现了去重——同一个 Perspective 被多个 Binding 引用时，Phase 1 用 Set 确保 Structure 只刷新一次。为了适配不同视角的差异化需求，我还把原本粗暴的 `refreshStructure: boolean` 开关重构为 `klStrategy` 策略：P23 日记视角用 `date-driven` 仅追加新日期的 KL，保护已有的 19 条记录不被覆盖；P01 方法论视角用 `synthesis` 全量重生成。这一改，系统终于懂得了“因视角而异”的智慧。

## 借鉴收集器模式：给脆弱的 Cron 穿上防弹衣

自动化跑通了，但新的焦虑来了：万一跑一半进程崩了怎么办？万一某个 Key Line 生成失败卡住整个队列怎么办？v1.3.0 版本的 `output_all` 就像个玻璃制品，无崩溃恢复、无重试机制、上下游无通信，一旦出错就得人工介入重头再来。

今天重头戏是引入 **inbox/batch 轮转机制**，直接照搬了 `js-knowledge-collector` 的生产消费模式。现在的流程变成了：`process_all` 在有变化时只往 `output-inbox.jsonl` 追加一行轻量信号；`output_all` 启动时先原子重命名 inbox 为 `output-batch-<ts>.json`，然后像流水线一样逐个处理。

最关键的是**逐步 Checkpoint**和**失败隔离**。以前是全部跑完才写状态，现在每处理完一个 Key Line 就更新一次 batch 文件里的 checkpoint。哪怕进程中途挂掉，重启后也能跳过已完成项，从断点续传。对于单个失败的 KL，系统不再“一人生病全家吃药”，而是将其标记为 `retry` 放入重试队列，自动重试不超过 3 次；若仍失败则标记 `permanently_failed` 并报警，同时不影响同批次其他 KL 的执行。

顺便修了个隐蔽的坑：原来配置 120 分钟间隔时，Cron 表达式直接生成 `*/120`，这在标准 Cron 里是非法的（分钟字段上限 59）。加了个 `minutesToCronExpr` 工具函数，自动把 `*/120` 转换为 `0 */2 * * *`，这种细节如果不处理，定时任务根本不会启动。

## 今天的收获

- 架构分层中，将强依赖的上下游环节整合为单一任务的内部阶段，比拆分为独立任务更能避免时序同步难题。
- 针对不同业务视角（如日记 vs 方法论）实施差异化的刷新策略（klStrategy），是保护既有数据资产的关键。
- 引入 inbox/batch 生产消费分离模式，能将脆弱的定时脚本升级为具备断点续传能力的高可用异步流水线。
- 逐步 Checkpoint 虽增加了文件 IO 次数，但将崩溃恢复的可靠性提升了一个数量级。
- 失败隔离配合有限重试（≤3 次）是平衡系统自愈能力与避免死循环的最佳实践。
