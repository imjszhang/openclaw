这里有个坑：路径里的反斜杠必须双写（`\\`），不然 JSON 解析会报错。我一开始直接复制了资源管理器里的路径，结果窗口重载后终端还是起不来，查了半天才发现是转义字符的问题。

配置保存后，执行 `Developer: Reload Window`。新建终端时，看到提示符从 `PS D:\...>` 变成了 `Administrator@DESKTOP-XXXXX MINGW64 /d/...$`，那一刻简直神清气爽。`ls -la`、`grep`、`cat` 这些命令终于能正常跑了，连 `openclaw` 的本地别名也能通过在 `~/.bashrc` 里加一行 `alias openclaw="pnpm --dir /d/github/fork/openclaw openclaw"` 轻松搞定。

## 企微部署的“生死 5 秒”：先 ACK 再干活

终端舒服了，接下来是重头戏：部署 WeCom 插件。企业微信的回调机制有个极其严苛的要求：**服务器必须在 5 秒内返回 HTTP 200**，否则它就会认为超时，然后重试，最多重试 3 次。这意味着如果你先解密消息、再丢给 AI 处理、等 AI 生成回复、最后再返回响应，大概率会超时。

我在写 `src/monitor.ts` 的回调处理逻辑时，特意把流程拆成了两步：

1.  **快速应答（ACK）**：收到 POST 请求后，验签通过，立刻返回 `200 "success"`。这一步必须在毫秒级完成，告诉企微“我收到了，你别重发了”。
2.  **异步处理**：在返回响应之后，再在后台解密 `Encrypt` 字段，解析 XML，把消息丢给 AI Agent 处理。

```typescript
// 先 ACK（企业微信要求 5 秒内响应，否则重试）
res.statusCode = 200;
res.end("success");

// 解密消息并异步处理（不阻塞 HTTP 响应）
const { msg: decryptedXml } = decryptWecom({ aesKey, cipherTextBase64: encrypt });
const msgObj = parseIncomingXml(decryptedXml);
// → 交给 processInboundMessage() 分发给 AI Agent
```
