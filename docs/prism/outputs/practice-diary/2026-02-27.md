## 渠道访问的“陷阱”：`open` 策略必须 paired 白名单

下午配置飞书渠道时踩了个逻辑陷阱。我想让机器人响应所有私聊消息，于是顺手把 `channels.feishu.dmPolicy` 设成了 `open`。本以为万事大吉，结果网关启动时报校验错误，死活起不来。

查阅文档才发现，`dmPolicy` 设为 `open` 是个强依赖条件：必须同时显式配置 `allowFrom: ["*"]`。系统逻辑是，既然你策略上开放了，就得在白名单里明确写上“所有人”，否则视为配置冲突。

```bash
pnpm openclaw config set channels.feishu.dmPolicy open
pnpm openclaw config set channels.feishu.allowFrom '["*"]'
```
