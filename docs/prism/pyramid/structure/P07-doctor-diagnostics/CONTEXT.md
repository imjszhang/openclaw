# P07 doctor diagnostics

## 读者

系统架构师、DevOps 工程师、核心维护者 | 核心诉求：寻求一套自动化机制，以最低成本保障系统长期健康状态，防止配置错误导致的生产事故

## SCQA 摘要

- **S**: OpenClaw 系统基于“本地优先网关”与五层抽象架构运行，其长期稳定性高度依赖配置文件的规范性与环境状态的一致性。随着系统迭代与人工干预的增加，配置漂移、路径解析不一致及潜在的安全隐患成为不可避免的常态。
- **C**: 传统的人工巡检与被动式故障修复不仅响应滞后，且难以覆盖深层的配置逻辑矛盾（如 State 与 Workspace 目录解析错位），导致系统在缺乏统一诊断防线的情况下，面临“小错积累成大祸”的不可维护风险。
- **Q**: 如何构建一种具备自动修复、深度审计及状态迁移能力的核心防线，以系统性保障 OpenClaw 的长期可维护性与配置规范化？
- **A**: 系统健康与配置规范化必须依赖自动化诊断工具（Doctor），将其打造为具备自动修复、深度审计及状态迁移能力的核心防线，以确保持续的可维护性。
