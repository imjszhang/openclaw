# KL13: JS ClawHub 轻量级生态导航站构建

> 所属视角：P23-practice-diary
> 上层论点：必须实施“架构分层分离 + 全链路自动化闭环 + 纵深安全防御”三位一体战略：以五层抽象网关为基石，通过双 Cron 分段驱动与 inbox/batch 轮转实现无人值守的知识产出，同时严格执行执行审批、权限收紧及记忆职责剥离，确保系统从“可记录”平稳演进为“可教学”的智能体。

## 支撑论点

### 13.1: 确立“纯静态架构 + 人工策展”的差异化定位，以零后端依赖实现高效聚合

- 逻辑顺序：结构
- 引用 atoms: JC-01, JC-02, JC-03, JC-04
- 引用 groups: G51

JS ClawHub 明确区别于官方 ClawHub 的安装发布功能，专注于解决“有什么值得关注”的策展发现需求。项目采用纯前端 JavaScript 技术栈，通过人工精选而非自动抓取，构建了轻量级且无后端依赖的生态导航核心。

### 13.2: 选用原生 HTML 与模块化 JS 构建极简技术栈，通过数据展示分离降低维护复杂度

- 逻辑顺序：结构
- 引用 atoms: JC-05, JC-06, JC-12, JC-13, JC-14
- 引用 groups: G51

摒弃 React/Vue 等重型框架，采用原生 HTML 配合 Tailwind CSS 及 Three.js 实现 Neo-Brutalism 风格，显著减少构建体积。架构上严格分离源码与部署产物，利用 JSON 管理数据并由模板渲染，实现了数据层与展示层的彻底解耦。

### 13.3: 打造六大核心功能模块并集成 AI 动态筛选，提供双语支持与社区热点实时聚合

- 逻辑顺序：结构
- 引用 atoms: JC-07, JC-08, JC-09, JC-10, JC-11
- 引用 groups: G51

系统涵盖项目导航、技能市场及 Pulse 动态等六大模块，其中 Pulse 利用 AI 每日自动过滤社交媒体噪音。通过自研轻量级 I18n 方案与 localStorage 持久化机制，实现了覆盖 UI 与内容的全站中英双语无缝切换。

### 13.4: 建立标准化 CLI 构建管线与自动化部署流程，实现从代码提交到 CDN 分发的全链路闭环

- 逻辑顺序：时间
- 引用 atoms: JC-15, JC-16, JC-17, JC-18, JC-19, JC-20
- 引用 groups: G51

通过 `cli/cli.js` 统一入口，执行 sync 命令即可触发包含文件复制、GA 注入及 API 生成等七步构建流程。该流程自动完成 Git 提交并推送至 GitHub Pages，配合 Cloudflare CDN 实现了首版快速上线及后续版本的持续迭代。

## 论点间关系

本 Key Line 遵循“战略定位 -> 技术架构 -> 功能实现 -> 工程落地”的结构顺序展开。首先明确项目作为 curated discovery 的核心价值，继而阐述支撑该价值的极简技术选型，随后展开具体的功能模块设计，最后以自动化的构建部署流程确保系统从概念到运行的完整闭环。
