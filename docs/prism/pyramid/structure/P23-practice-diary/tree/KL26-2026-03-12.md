# KL41: 模板创建技能需采用“决策引导 + 确定性 Scaffold+ 渐进式披露”策略以降低多层知识门槛

> 所属视角：P23-practice-diary
> 上层论点：必须实施“架构分层分离 + 全链路自动化闭环 + 纵深安全防御”三位一体战略：以五层抽象网关为基石，通过双 Cron 分段驱动与 inbox/batch 轮转实现无人值守的知识产出，同时严格执行执行审批、权限收紧及记忆职责剥离，确保系统从“可记录”平稳演进为“可教学”的智能体。

## 支撑论点

### 41.1: 针对模板创建的高门槛痛点，应设计专用技能引导而非仅提供 CLI，以释放 LLM 在创意内容生成上的核心价值

- 逻辑顺序：结构
- 引用 atoms: PT-01, PT-02
- 引用 groups: G61

模板创建涉及多层知识导致新用户门槛极高，单纯提供 CLI 工具无法解决认知负担问题。LLM 的核心优势在于生成高质量的创意内容（如 Prompt 编写），而非仅生成空骨架，因此需通过专用技能将用户从结构配置中解放出来。

### 41.2: 技能架构应采用“决策引导 +Schema 参考+Scaffold 工具”三层设计，通过四问映射实现配置自动化

- 逻辑顺序：结构
- 引用 atoms: PT-03, PT-04
- 引用 groups: G61

`prism-template-author`技能通过 SKILL.md 进行决策引导，利用四个关键问题（产出形态、写作风格、生成复杂度、质量审校）自动映射到具体的 split、type 和组件配置。这种设计将复杂的配置逻辑封装在引导流程中，配合 Schema 参考文件，确保用户只需关注业务意图而非技术细节。

### 41.3: 核心工具链须严格遵循“纯确定性逻辑构建骨架，Agent 负责内容填充”的分工策略

- 逻辑顺序：程度
- 引用 atoms: PT-05, PT-06, PT-08
- 引用 groups: G61

`prism_scaffold_template` 和 `prism_scaffold_component` 工具被设计为纯确定性逻辑，仅负责校验模板名、合并默认值、生成 frontmatter 及区段占位，绝不调用 LLM。这种严格的职责剥离确保了文件结构的绝对正确性，而将具体的 Persona 或 Style 等内容填充任务完全交给 Agent 根据技能引导完成。

### 41.4: 实施渐进式披露策略与标准化路径规范，确保技能文件轻量化及产出物的一致性

- 逻辑顺序：结构
- 引用 atoms: PT-07, PT-09, PT-10, PT-12
- 引用 groups: G61

主技能文件控制在 130 行以内，详细 Schema 置于附件供按需查阅，有效降低认知负荷。同时，规范了文件写入路径（优先 outputs/\_templates/）及生成参数（name, split, type, stages），并明确了包含 SKILL.md 及 openclaw-plugin 的标准目录结构，保证产出物的可维护性。

### 41.5: 建立基于 dry-run 的验证机制，确保新生成模板在写入前的结构正确性

- 逻辑顺序：时间
- 引用 atoms: PT-11
- 引用 groups: G61

在模板正式落盘前，强制通过 `npx js-knowledge-prism output --dry-run` 命令进行预验证。这一步骤作为自动化闭环中的关键质检点，能够在不污染知识库的前提下确认新生成模板的逻辑与结构是否符合预期。

## 论点间关系

本 Key Line 遵循“问题定义 -> 架构设计 -> 核心策略 -> 规范落地 -> 验证闭环”的结构顺序。首先明确高门槛痛点与 LLM 价值定位，随即提出三层技能架构与四问映射方案；接着确立确定性工具与 Agent 创意填充的分工边界，并辅以渐进式披露和路径规范确保执行细节；最后通过 dry-run 验证机制形成完整的质量闭环。
