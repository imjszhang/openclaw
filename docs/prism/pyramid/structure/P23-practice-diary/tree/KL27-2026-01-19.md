# KL-11: JS Eyes 逆向架构模式

> 所属视角：P23-practice-diary
> 上层论点：必须实施“架构分层分离 + 全链路自动化闭环 + 纵深安全防御”三位一体战略：以五层抽象网关为基石，通过双 Cron 分段驱动与 inbox/batch 轮转实现无人值守的知识产出，同时严格执行执行审批、权限收紧及记忆职责剥离，确保系统从“可记录”平稳演进为“可教学”的智能体。

## 支撑论点

### 11.1: 采用“浏览器主动连接 + 逆向架构”填补 Agent 框架在实时复用用户环境上的能力空白

- 逻辑顺序：结构
- 引用 atoms: EP-01, EP-02, EP-03
- 引用 groups: G64

主流 Agent 框架依赖独立进程启动浏览器，无法轻量级实时复用用户当前环境；JS Eyes 通过扩展主动连接 WebSocket 服务器，构建了由浏览器扩展、WS 服务器、Agent 插件组成的三层解耦架构，实现了对用户正在使用浏览器的低侵入实时控制。

### 11.2: 构建跨平台核心能力闭环并遵循“功能→安全→稳定→生态”的迭代演进路径

- 逻辑顺序：时间
- 引用 atoms: EP-04, EP-05, EP-07, EP-09, EP-10

系统适配 Chrome (MV3) 与 Firefox (MV2) 实现功能对等，具备实时通信、标签同步及断线指数退避重连等五大核心能力；项目演化严格遵循从 v1.1.0 核心功能闭环，到 v1.2.0 引入 HMAC-SHA256 认证，再到 v1.3.x 专注速率限制与熔断机制的稳健迭代逻辑。

### 11.3: 基于 OpenClaw 插件注册表机制实现技能生态的自动化发现、安装与一键部署

- 逻辑顺序：程度
- 引用 atoms: EP-11, EP-13, EP-14, EP-15

在基础自动化之上，通过 OpenClaw 插件注册 9 个 AI 工具（涵盖 HTML 获取、脚本执行等）形成高层技能生态；支持 Agent 自动调用 discover/install 接口查询注册表并完成下载注册，亦支持通过 curl 管道或 PowerShell 变量赋值进行一键式技能部署。

## 论点间关系

子论点按“架构设计原理（解决连接痛点）→ 核心能力演进（确保稳定安全）→ 生态应用扩展（实现自动化部署）”的结构逻辑展开，层层递进地构建了从底层连接到上层智能应用的完整技术闭环。
