## 与 synthesis 的差异记录

本视角以**时间线**为主轴，synthesis 以**主题**为主轴，存在结构性差异：单个 KL 天内可能覆盖多个 synthesis 候选，一个 synthesis 候选也可能跨越多个 KL。

| 差异点                               | synthesis 中的观点                   | tree 中的处理                                                          | 调整原因                                                                                       |
| ------------------------------------ | ------------------------------------ | ---------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| S1–S3（知识方法论）分散到多个 KL     | S1/S2/S3 是并列的顶层方法论观点      | 分布在 KL09（知识库设计）、KL10（棱镜方法论）、KL11–KL12（工具化实践） | 时间线叙事需按事件顺序展开，无法像 synthesis 那样一次集中呈现                                  |
| S6（Fork 管理）首尾呼应              | S6 是单一主题候选                    | 分布在 KL01（首日决策）和 KL09/KL15（两次大合并实践）                  | 时间线视角下 Fork 管理是贯穿全程的主线，"首次决策→第一次实战→第二次成熟"的演进弧线更有叙事张力 |
| S22（知识棱镜插件化）与 S1–S3 的关系 | synthesis 中 S22 是 S1–S3 的实践闭环 | KL11.2 作为"爆发日"的一个支撑点，而非单独 KL                           | 插件化是当天多件事之一，日记叙事重心在"角色转变"而非"闭环"概念，不宜单独强调                   |
| S28–S30（权限/记忆/配置边界）        | S28/S29/S30 是独立的保障层候选       | 分布在 KL13（权限）、KL14（记忆）、KL12（exec 安全）                   | 日记视角这三件事发生在不同天，按时间自然分布；synthesis 的主题归纳是后验总结，不影响 tree 结构 |
