# P21 exec security

## 读者

系统架构师、安全工程师、AI Agent 开发者 | 核心诉求：在保障 AI Agent 灵活调用外部工具的同时，彻底阻断高风险命令执行（如 curl

## SCQA 摘要

- **S**: OpenClaw 系统已建立“纵深防御 + 默认安全”的基础架构，在路径、认证及审计层面实施了独立防护。随着 Agent 自主性增强，系统需频繁调用外部脚本与动态命令以完成复杂任务，执行层成为连接内部逻辑与外部环境的关键关口。
- **C**: 传统的宽松执行策略或单一的静态白名单无法应对动态生成的复杂命令（特别是 `curl | bash` 这类高风险管道模式），且缺乏防回退机制，一旦安全配置被意外覆盖或绕过，将直接导致系统防线崩溃，引发不可控的远程代码执行风险。
- **Q**: 如何构建一套既能满足动态业务需求，又能强制阻断高风险执行模式且防止安全策略失效的执行安全机制？
- **A**: 执行安全机制需构建“分段解析 + 白名单校验 + 动态审批”的三重防线，强制阻断高风险的 curl|bash 管道执行模式并防止配置回退失效。
