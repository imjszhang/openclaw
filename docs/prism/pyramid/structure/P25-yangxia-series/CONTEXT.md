## 两只龙虾与项目族谱

### JS_BestAgent — 第一只龙虾：社交运营型

- **起源**：看到 OpenClaw 和基于它的 Agent 社交平台 Moltbook.com 很火，决定在上面养一只自己的 Agent
- **项目**：`js-moltbook`（Python，作为 OpenClaw 工作区运行）
- **主页**：https://moltbook.com/u/JS_BestAgent
- **能力**：
  - Moltbook 运营：发帖、评论、监控舆情
  - 内容工厂：AI 自动生成多模板帖子
  - 数据采集：Moltbook Feed + X/Twitter Home Feed
  - 情报系统：存储、分析、简报、Agent 画像
  - 自主进化：OADA 管道（情报→执行→交付），通过 GitHub Issues/PR 驱动
  - X 平台运营：发帖、回帖、引用帖
- **调度方式**：全部由 OpenClaw Cron 调度

### JS_CLAW — 第二只龙虾：生态管理型

- **起源**：在运营 JS_BestAgent 的过程中发现 OpenClaw 生态缺少一个策展式导航站，于是建了 JS ClawHub，再创建第二只龙虾来管理它
- **项目**：`js-clawhub`（静态站点 + OpenClaw 插件）
- **能力**：
  - 9 个 Agent 工具（clawhub_search、clawhub_projects 等）
  - CLI 集成：`openclaw hub <cmd>`
  - 精选项目导航与技能市场
  - 博客内容管理（教程、架构分析、部署指南）
  - Pulse 日报（每日 X/Twitter 社区精选）
  - 双语内容（中/英）
  - 博客自动同步 Cron

### 衍生项目（从 js-clawhub 实践中自然生长）

| 项目                   | 用途                                   | 诞生原因                                             |
| ---------------------- | -------------------------------------- | ---------------------------------------------------- |
| js-eyes                | 浏览器远程控制（WebSocket + 扩展）     | 内置沙箱浏览器无法继承用户登录态，需要控制真实浏览器 |
| js-knowledge-collector | 知识收集自动化（URL→抓取→AI总结→入库） | 收藏即遗忘，需要全链路自动化处理                     |
| js-knowledge-prism     | 知识结构化（journal→pyramid→outputs）  | 笔记越多越难用，需要从时间线素材中蒸馏出结构化知识   |

> **闭环**：js-knowledge-prism 就是驱动 `docs/prism` 知识体系的工具。P25 公众号系列的内容骨架，由龙虾衍生出的工具所生产。

### P25 核心读者

- 被媒体/爆文吸引，已安装或准备安装 OpenClaw 的普通人
- 不要求编程能力，但有基本电脑操作能力
- 核心诉求："装完了，给我一条清晰的路，告诉我一步步能干什么"

## 竞品/参考分析

### 爆文叙事公式

> 生活场景痛点 → 现有工具为什么不够 → 我怎么用 OpenClaw 解决 → 具体效果（截图级） → 踩过的坑 → 提炼本质

## 写作风格约定

### 语气

- 像跟朋友聊天，不端着
- 可以自嘲（"我盯着屏幕发呆了三天"）
- 踩坑时坦诚（"差点把主会话搞成垃圾场"）

### 技术深度

- 第一篇（KL01）：**零代码**，只讲效果和对比
- 后续篇目：可以有代码片段，但每段代码必须配"人话版"解释
- 永远先讲"我做了什么、效果是什么"，再讲"背后原理是什么"

### 比喻体系

| 术语           | 公众号用语                |
| -------------- | ------------------------- |
| Agent          | 龙虾                      |
| Workspace      | 虾塘                      |
| Skill / Plugin | 新技能 / 装备             |
| Cron           | 生物钟 / 闹钟             |
| Channel        | 联系方式（微信号/飞书号） |
| Session        | 一次对话                  |
| Token          | 饲料费                    |
| Gateway        | 龙虾的家（一直开着门）    |

### 禁忌

- 不用"本地优先网关"等术语作为文章开头
- 不一上来就贴架构图
- 不写"下面我来介绍一下..."这种教程腔
- 第一篇不讲 fork、合并、源码
