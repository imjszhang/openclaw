### 4.2: 第 1-3 条——OpenClaw 是什么 + 准备清单 + API Key 申请

**素材要点**（来自 2 万字教程第 1-3 章）：

- **#1 OpenClaw 是啥**：住在你电脑里的 AI 助理，不是 ChatGPT 替代品，是平台
- **#2 它能做什么**：三个真实场景（自动整理日报、查资料写报告、飞书里@它办事）
- **#3 准备清单**：一台能上网的电脑、一个 API Key、10 分钟时间
- **API Key 申请**：国内三家 Coding Plan（KIMI/MiniMax/GLM）的详细步骤
- **关键提醒**：Key 创建后只显示一次，丢了只能重新创建，像银行卡密码一样保管好

**支撑 Groups**：G12（话题 slug）

### 4.3: 第 4-8 条——安装流程（Node.js→安装命令→向导配置）

**素材要点**（来自 2 万字教程第 3 章）：

- **#4 Node.js 检查**：`node --version`，需要 v22+，Mac/Windows/Linux 安装命令
- **#5 安装命令**：`npm install -g openclaw@latest`，就一行
- **#6 运行向导**：`openclaw onboard --install-daemon`
- **#7 模式选择**：QuickStart（新手）vs Manual（老手），建议 QuickStart
- **#8 模型配置**：选厂商→选鉴权方式→输入 API Key→选模型
- **关键提醒**：Channel 先 Skip，等 TUI 对话通了再接

**支撑 Groups**：G12（话题 slug）

### 4.4: 第 9-13 条——飞书接入（创建应用→权限配置→长连接时序）

**素材要点**（来自 2 万字教程第 5 章）：

- **#9 飞书开放平台**：创建企业自建应用，获取 App ID 和 App Secret
- **#10 权限配置**：批量导入权限 JSON（15 个权限）
- **#11 关键时序**：先发布应用→再开长连接（血的教训）
- **#12 配对放行**：私聊机器人获取配对码，`openclaw pairing approve feishu <CODE>`
- **#13 安全第一课**：私聊策略选 pairing（推荐）vs allowlist vs all（危险）
- **关键提醒**：时序错了必挂，这是时序问题，不是运气问题

**支撑 Groups**：G12（话题 slug）

### 4.5: 第 14-17 条——安全配置 + 模型选择

**素材要点**（来自 2 万字教程第 6-7 章）：

- **#14 群聊保险栓**：requireMention 必须开，避免机器人乱回消息
- **#15 上线前 checklist**：5 个检查项（私聊策略、群聊策略、白名单、预算上限、高风险工具）
- **#16 模型怎么选**：三家各有脾气，先用已开通的那家跑通
- **#17 配 fallback**：主力挂了自动切备胎，无感切换
- **关键提醒**：过完 5 条 checklist 再放出去，不过完别放
