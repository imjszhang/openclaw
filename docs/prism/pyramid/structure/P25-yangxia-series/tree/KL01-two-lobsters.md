# KL01: 我养了两只 AI 龙虾——通过它们的日常，看 OpenClaw 能干什么

> 所属视角：P25 - JS的养虾日记（公众号系列）
> 上层论点：塔尖

## 核心论点

不讲概念、不做对比、不贴代码——直接让读者认识 JS_BestAgent 和 JS_CLAW 这两只龙虾，看看它们每天在干什么。读者看完自然理解"OpenClaw 能干什么"，比任何功能列表都有说服力。

## 叙事结构

本篇不采用"论点-论据"的说理结构，而是采用**故事弧线**：人物出场→日常展示→自然演化→衍生项目→闭环点题→系列预告。

### 1.1: 开场钩子——龙虾们的"今天"

- 逻辑顺序：时间（从一个具体的"今天"开始）
- 来源 journal：js-moltbook 项目实际运行数据, [clawhub-blog-auto-sync-cron.md](../../../../../journal/2026-03-08/clawhub-blog-auto-sync-cron.md)
- 引用 groups：G50（多库自动化）, G53（博客同步）
- P23 对应：KL18（Day 36）, KL19（Day 37）

**素材要点**：

- JS_BestAgent 今天的实际输出：在 Moltbook 发帖、评论互动、X/Twitter 采集
- JS_CLAW 今天的实际输出：更新 ClawHub Pulse 日报、同步博客、精选项目

**公众号写法**：用"日报"式开场，直接展示两只龙虾今天干了什么。不解释它们是什么，先让读者看到效果。

> 示例方向："JS_BestAgent 今天又在 Moltbook 上发了三条帖子，还顺便跟别人的 Agent 争论了一番。JS_CLAW 默默更新了 ClawHub 的今日精选，把最新的社区热点同步到了博客里。这两只龙虾是我养的。它们每天自己干活，不需要我盯着。"

### 1.2: 第一只龙虾——JS_BestAgent 的诞生和日常

- 逻辑顺序：时间（从"为什么养它"到"它每天在干什么"）
- 来源 journal：js-moltbook 项目 README / workflow.py, [cron-config-guide.md](../../../../../journal/2026-02-05/cron-config-guide.md)
- 引用 groups：G21（Cron 调度）, G18（Agent 自我进化）
- 引用 synthesis：S15（Cron）, S12（Agent 进化）
- P23 对应：KL03（Day 5）, KL04（Day 6）

**素材要点**：

- 起源故事：看到 OpenClaw 和 Moltbook（Agent 社交平台）很火 → 想在上面养一只自己的 Agent
- JS_BestAgent 的一天（用"时间线"展示）：
  - 早上：扫描 Moltbook 新帖，挑有价值的评论
  - 上午：根据内容模板自动生成并发布帖子
  - 下午：采集 X/Twitter 上的 OpenClaw 相关讨论
  - 晚上：跑情报分析管道，生成简报
  - 持续：OADA 自我进化——回顾哪些帖子效果好，调整策略
- 关键信息：这一切都是 Cron 定时任务驱动的，全自动运行

**公众号写法**：用"一天的时间线"让读者看到一只龙虾的完整日常。自然带出"它不是聊天机器人，是一个 7×24 在线的自动化Agent"。ChatGPT 对比不需要单独一节，一句话带过即可："ChatGPT 你关了网页它就消失了。JS_BestAgent 我睡着了它还在社交。"

### 1.3: 从第一只到第二只——为什么会有 JS_CLAW

- 逻辑顺序：时间（自然演化的叙事）
- 来源 journal：[js-clawhub-project-creation](../../../../../journal/2026-02-12/js-clawhub-project-creation.md)
- 引用 groups：G51（JS ClawHub 策展导航）
- 引用 synthesis：S32（生态导航站）, S35（ClawHub 插件化）
- P23 对应：KL19（Day 37）

**素材要点**：

- 玩 JS_BestAgent 的过程中发现 OpenClaw 生态缺一个策展式导航站
- 建了 JS ClawHub：精选项目、技能市场、博客、Pulse 日报、双语
- 手动维护太累 → 养了第二只龙虾 JS_CLAW 来管它
- JS_CLAW 的能力：9 个 Agent 工具、CLI 集成、博客自动同步 Cron

**公众号写法**：讲"需求是怎么自然冒出来的"。不是规划出来两只龙虾，是第一只养着养着，自然就需要第二只了。这个演化过程本身就在展示 OpenClaw 的扩展能力。

### 1.4: 龙虾的孩子们——三个衍生项目（点到为止）

- 逻辑顺序：结构（三个项目并列展示）
- 来源 journal：[js-eyes-openclaw-plugin-guide.md](../../../../../journal/2026-02-25/js-eyes-openclaw-plugin-guide.md), [js-knowledge-collector-project-creation.md](../../../../../journal/2026-03-06/js-knowledge-collector-project-creation.md), [from-notes-to-plugin.md](../../../../../journal/2026-02-25/from-notes-to-plugin.md)
- 引用 groups：G34（JS-Eyes）, G52（知识收集器）, G33（知识棱镜）
- 引用 synthesis：S23（JS-Eyes）, S33（知识收集自动化）, S22（知识棱镜自动化）
- P23 对应：KL06（Day 11）, KL11（Day 25）, KL17（Day 35）, KL10（Day 24）

**素材要点**：

- **js-eyes**：龙虾需要操控真实浏览器（继承登录态） → 开发了浏览器控制扩展
- **js-knowledge-collector**：龙虾需要自动收集知识（URL→抓取→AI总结→入库） → 开发了全链路收集器
- **js-knowledge-prism**：笔记越来越乱 → 开发了把散乱笔记变成结构化知识的工具
- 每个项目都是上一步实践中暴露的具体痛点催生的

**公众号写法**：每个项目用一句话讲"为什么需要它"和"它能干什么"，不展开技术。为后续篇目埋钩子。

### 1.5: 闭环点题——回答"OpenClaw 能干什么"

- 逻辑顺序：收敛
- 引用 synthesis：S9（核心架构——降级为一句话）

**素材要点**：

- 闭环揭示：js-knowledge-prism 就是整理这篇文章内容骨架的工具 → "你正在读的这篇文章，就是龙虾帮我整理出来的"
- 回答核心问题：OpenClaw 能干什么？看看这两只龙虾和它们的三个孩子——一只替我社交，一只替我管网站，还衍生出了浏览器控制、知识收集和知识结构化三个工具
- 一句话定位差异（非单独一节，自然带出）：它不是更好的 ChatGPT，是一个能养出"活物"的平台

**公众号写法**：短而有力的收尾。用闭环制造"哇"感，用预告制造追更动力。

## 论点间关系

本篇是**叙事弧线**而非论证结构：

```
1.1 钩子（展示效果）
  → 1.2 第一只龙虾（人物+日常）
    → 1.3 第二只龙虾（演化故事）
      → 1.4 衍生项目（生态扩展）
        → 1.5 闭环点题（回答核心问题）
          → 1.6 预告（制造追更）
```

读者的认知轨迹：

1. "哦？AI 还能自己发帖？"（好奇）
2. "这只龙虾一天干这么多事？"（惊讶）
3. "还能养第二只来管网站？"（兴趣加深）
4. "还衍生出了三个工具？"（震撼）
5. "这篇文章本身就是龙虾的产出？"（闭环感）
6. "我也想养一只！"（追更动力）
