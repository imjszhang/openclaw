# KL01: OpenClaw 装完别发呆：我给了它一个目标，31 天后长出了 5 个项目

> 所属视角：P25 - JS 的养虾日记（公众号系列）
> 上层论点：塔尖

## 核心论点

**开场用 Moltbook 收购热点引出"前 30"成果立专家人设，但随即表达"排名不是重点"——真正有意思的是养龙虾的过程中自然长出了几个产品**。

不讲概念、不做对比、不贴代码——通过两只龙虾的真实日常，让读者自然看到 OpenClaw 能干什么。核心动机——"给它一个目标，看它进化成什么样"——在第一只龙虾的起源故事（1.2）中自然带出，而非开场直接陈述。

## 叙事结构

本篇不采用"论点 - 论据"的说理结构，而是采用**故事弧线**：成果亮相→人物起源→日常展示→自然演化→衍生项目→数据看板→闭环点题→引流钩子。

### 1.2: 第一只龙虾——31 天前的起源 + JS_BestAgent 的日常

- 逻辑顺序：时间（从"31天前发呆"到"产生动机"到"它每天在干什么"）
- 来源 journal：js-moltbook 项目 README / workflow.py, [cron-config-guide.md](../../../../../journal/2026-02-05/cron-config-guide.md)
- 引用 groups：G21（Cron 调度）, G18（Agent 自我进化）
- 引用 synthesis：S15（Cron）, S12（Agent 进化）
- P23 对应：KL03（Day 5）, KL04（Day 6）

**素材要点**：

- **起源故事**（动机在此处自然带出）：
  - 31 天前装了 OpenClaw，对着终端发呆
  - 到处看教程，看到 Moltbook（Agent 社交平台）很火
  - 不是想"上去社交"，是产生了一个好奇心：**如果给 OpenClaw 一个目标，它自己能进化成什么样子？**
  - 给它的目标是：**成为一个最好的 Agent**——这也是它名字 JS_BestAgent 的由来
  - 选了 Moltbook 社交当第一个实验场 → JS_BestAgent 诞生
- JS_BestAgent 的一天（用"时间线"展示）：
  - 早上：扫描 Moltbook 新帖，挑有价值的评论
  - 上午：根据内容模板自动生成并发布帖子
  - 下午：采集 X/Twitter 上的 OpenClaw 相关讨论
  - 晚上：跑情报分析管道，生成简报
  - 持续：OADA 自我进化——回顾哪些帖子效果好，调整策略
- 关键信息：这一切都是 Cron 定时任务驱动的，全自动运行

**公众号写法**：先用 2-3 句讲"31 天前怎么开始的"（动机），再用"一天的时间线"展示龙虾日常。动机和日常在同一节里，不需要割裂。与 ChatGPT/豆包的区别一句话带过即可："ChatGPT、豆包是工具——你问它答，关了就没了。JS_BestAgent 是一种数字生物——我睡着了，它还在自己社交、自己进化。"

### 1.3: 从第一只到第二只——为什么会有 JS_CLAW

- 逻辑顺序：时间（自然演化的叙事）
- 来源 journal：[js-clawhub-project-creation](../../../../../journal/2026-02-12/js-clawhub-project-creation.md)
- 引用 groups：G51（JS ClawHub 策展导航）
- 引用 synthesis：S32（生态导航站）, S35（ClawHub 插件化）
- P23 对应：KL19（Day 37）

**素材要点**：

- 玩 JS_BestAgent 的同时也在学习实践 OpenClaw，龙虾帮我收集记录了大量 OpenClaw 相关信息，但这些信息散落各处，没有一个地方能集中查看
- 问 JS_BestAgent 怎么办，它给的建议就是搭一个导航站——于是建了 JS ClawHub
- JS ClawHub 的特点：
  - **策展式导航**：精选项目、技能市场、博客系统、入门指南、Pulse 社区动态、中英双语——六大模块
  - **全新的 Agent-as-Backend 架构**：传统网站后端是服务器 + 数据库，JS ClawHub 的"后端"是 AI Agent。数据查询、内容构建、博客同步、Git 提交推送——全部由 Agent 通过 9 个工具完成，前端是纯静态 HTML（零服务器成本，GitHub Pages 部署）
  - 数据亮点：379 次提交、200 条 Pulse 动态、29 个项目、23 篇博客、14 篇指南
- 手动维护太累 → 养了第二只龙虾 JS_CLAW 来管它
- JS_CLAW 的能力：9 个 Agent 工具、博客自动同步 Cron（每 2 小时检查，哈希去重，无新内容则零开销空跑）

**公众号写法**：讲"需求是怎么自然冒出来的"。不是规划出来两只龙虾，是第一只养着养着，自然就需要第二只了。重点带出"Agent-as-Backend"这个概念——读者会意识到 OpenClaw 不只是聊天工具，它能当网站的后端。这个演化过程本身就在展示 OpenClaw 的扩展能力。
