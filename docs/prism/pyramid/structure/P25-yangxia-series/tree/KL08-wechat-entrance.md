## 四层支撑结构

### 25.8.1 安装实测：一行指令 + 扫码配置

**核心观点**: 腾讯把门槛降到最低，但"低门槛"不等于"要托管"

**素材要点**:

- 官方指令：`npx -y @tencent-weixin/openclaw-weixin-cli@latest install`
- 配置门槛：微信 8.0.70+，扫码配置
- 隐私机制：私密通道，一对一私聊，他人无法添加
- 功能现状：仅支持私聊，暂无法进群（后续可能支持）

**引用来源**:

- 微信文章：《微信原生支持各种 OpenClaw 龙虾接入，附教程》

### 25.8.2 隐私边界：私密通道 vs 云端托管

**核心观点**: 微信是通道，数据在你手里——这才是正确的打开方式

**素材要点**:

- 微信 ClawBot 定位：消息收发通道，不自动化操作账号
- 数据流向：你的终端 ↔ 微信 ↔ 你的龙虾（腾讯云不碰数据）
- 对比 KL02 四层架构：
  - 配置透明：你的龙虾，配置文件在你手里
  - 响应速度：本地即时生效
  - 防御深度：四层纵深防御 intact
  - 信任边界：你是主人，不是分母

**引用来源**:

- KL02-master-not-fear.md（四层对比框架）
- G07（纵深防御）, G40（权限体系）, G43（权限指南）

**关键金句**:

- "腾讯想要的是入口，你想要的是主权"
- "微信是通道，龙虾是你的——这才是正确的打开方式"
- "通道可以借，主权不能让"

---

### 25.8.3 生态卡位：腾讯的 AI 操作系统野心

**核心观点**: 这不是功能更新，是生态位争夺战

**素材要点**:

- 腾讯的四张牌：
  1. **入口优势**：12 亿月活，覆盖所有中国人的社交/支付/工作场景
  2. **生态整合**：AI Agent 作为"超级胶水"串联公众号/小程序/支付/视频号
  3. **竞争压力**：字节飞书 + 通义、钉钉、Telegram Bot 生态
  4. **商业变现**：ToC 免费入口 → ToB 云服务（GPU/混元 API/存储）

- 腾讯的漏斗模型：
  - 顶层：免费微信入口吸引海量用户
  - 中层：大模型 API 调用量
  - 底层：企业级方案（GPU 算力/对象存储/CDN）

**引用来源**:

- 微信文章战略分析部分
- KL07-app-disappears.md（App 消亡论）

**关键金句**:

- "Pony 这一手不是在做一个功能，是在卡一个生态位"
- "微信 + AI Agent 这个组合如果跑通了，就是下一代超级应用的雏形"
- "OpenClaw 只是那把打开门的钥匙——门后面是整个腾讯生态"
