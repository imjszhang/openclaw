# KL02: 掌控 AI，而不是畏惧 AI——云端 vs 本地，四个层面看懂为什么必须本地部署

> 所属视角：P25 - JS 的养虾日记（公众号系列）
> 上层论点：塔尖

## 核心论点

**核心观点**：OpenClaw 要装在工作机（本地机）里才最安全——不是云端，不是服务器，是你每天用的这台电脑。

**洞察**：一切的恐惧，来自于不懂。CNCERT 风险提示吓坏了很多人，但 13.5 万台设备遭殃的根本原因，不是 OpenClaw 本身不安全，是用户不懂怎么配置、怎么审计、怎么控制。

**干货**：本文给你一套完整的"工作机安全部署指南"——从权限配置到审计日志，从 CNCERT 建议落地到 31 天实战验证。

核心对立：**云端托管（黑箱/被动/单点/多租户）vs 本地部署（透明/主动/纵深/个人模型）**。

### 【承】默认配置不安全 + 云端无法调配置

- 逻辑顺序：问题→原因→对比
- 来源 journal：权限配置指南、exec 审批机制文档、CNCERT 风险提示
- 引用 groups：G40（权限体系）, G43（权限指南）, G39（exec 审批）

**素材要点**：

**第一部分：默认配置为什么不安全？**

- CNCERT 原文："默认的安全配置极为脆弱"
- 四大风险：提示词注入、误操作、插件投毒、漏洞危害
- 根本原因：默认配置下，权限过高 + 无审计 + 无隔离

**第二部分：云端托管为什么不能调配置？**

- 配置是黑箱——你不知道厂商的 `openclaw.json` 和 `exec-approvals.json` 里配了什么
- 工具可见性与执行权限混为一谈——云端通常只给一个"开/关"开关
- 出问题不知道查哪个配置文件——因为根本看不见

**第三部分：本地部署为什么能调配置？**

- 两个配置文件摊在你面前：`openclaw.json`（意图）和 `exec-approvals.json`（运行时）
- 权限双轨制清晰独立：工具可见性（allow/deny）与 exec 执行权限各自配置
- 你能看懂每一行配置是什么意思，知道哪里能改、怎么改、改了会怎样

**公众号写法**：

1. 用"CNCERT 原文 + 四大风险"说明默认配置不安全
2. 用"云端配置是黑箱"说明无法调配置
3. 用"本地两个 JSON 文件"说明能调配置
4. 过渡到转："但光能调配置还不够，关键是什么？"

**生活中的例子**：

1. **小孩怕黑**——不是怕黑暗本身，是怕"不知道黑暗里有什么"。开灯就不怕了，因为看懂了。

2. **老人怕微波炉**——不是微波炉真的危险，是不懂原理。知道微波是什么、怎么被屏蔽的，就不怕了。

3. **坐飞机怕颠簸**——不懂的人觉得"飞机要掉下去了"，懂的人知道"这只是气流，飞机设计能承受比这强十倍的颠簸"。

4. **第一次开车上高速**——新手手心出汗，怕出事。开了三个月后，同样的路，同样的车，心态完全不同——因为懂了。

5. **看体检报告**——看到指标异常就慌。医生一说"这个没事，这个要注意"，恐惧就没了——因为懂了。

**OpenClaw 是一样的道理**：

- CNCERT 说"默认配置脆弱"——你慌了，因为不懂
- 看完这篇文章，知道怎么配、怎么审计、怎么控制——恐惧变成掌控感
- 31 天，零事故——不是运气，是懂了之后的自然结果

**干货：工作机安全部署指南（5 步）**

**第一步：安装前的准备**

1. 确认工作机是"你的"电脑（不是公司域管电脑）
2. 准备独立的 API Key（不要用公司的）
3. 准备密码管理器（1Password/Bitwarden）

**第二步：安装时的配置**

1. 选择 QuickStart 模式（先跑通，再优化）
2. Channel 先 Skip（等 TUI 对话通了再接）
3. 完成 bootstrap 对话（把 Agent 变成"你的"Agent）

**第三步：安全配置（CNCERT 四项建议落地）**

1. **网络隔离**：不将管理端口暴露公网，使用容器限制权限
2. **凭证管理**：API Key 存在密码管理器，不存微信/不截图
3. **插件管控**：禁用自动更新，手动审核每个 Skill 源码
4. **补丁更新**：关注官方安全动态，手动验证后再升级

**第四步：权限审计（三层防线）**

1. **私聊策略**：选 pairing（用户申请，管理员批准）
2. **群聊保险栓**：requireMention 必须开（只回复@消息）
3. **高风险工具**：execute_command、send_email 关掉或加二次确认

**第五步：上线前 checklist**

- [ ] 私聊策略是 pairing 或 allowlist，不是 all
- [ ] 群聊 requireMention 已开启
- [ ] 群白名单只允许必要的群
- [ ] 预算上限已设置（`openclaw config set budget.monthly 50`）
- [ ] 高风险工具已限制或关闭
