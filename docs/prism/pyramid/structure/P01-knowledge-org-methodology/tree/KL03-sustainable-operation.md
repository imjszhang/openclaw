# KL03: 文档化流程、模板化入口和可追溯链路保障拆解体系持续运转

> 所属视角：P01 - 个人知识库的结构化方法
> 上层论点：塔尖

## 支撑论点

### 3.1: 增量拆解流程文档化——每次新增都有章可循

- 逻辑顺序：时间（按执行步骤排列）
- 引用 atoms: KA-16
- 引用 groups: G06

journal 持续增长意味着拆解永远不是一次性完成的。文档化的增量流程（新 journal → atoms → groups → synthesis → 各视角 → 修订记录）确保每次新增都能按同一套路径执行（KA-16），降低认知负担和遗漏风险。

### 3.2: 视角模板化——降低新建成本，保持格式一致

- 逻辑顺序：结构（机制描述）
- 引用 atoms: KA-17
- 引用 groups: G06

提供 `_template/` 目录作为新视角的快速起步入口（KA-17）。复制模板改名即可获得完整的 scqa.md + tree/ + validation.md 骨架，既降低了新建成本，又保障了所有视角格式统一、结构可比。

### 3.3: 五项设计原则统领所有决策方向

- 逻辑顺序：结构（原则总括）
- 引用 atoms: KA-18
- 引用 groups: G06

职责分离、扩展性优先、共享与隔离、增量友好、可追溯（KA-18）——这五条原则既是架构决策的事后总结，也是未来演化时的决策指南。当遇到新的组织困惑时，回到原则层面验证即可。

## 论点间关系

3.1→3.2→3.3 构成结构顺序：先定义"做什么"（增量流程），再解决"怎么做得轻松"（模板化），最后回答"为什么这样做"（设计原则）。从执行层到机制层到原则层，逐步抽象。
