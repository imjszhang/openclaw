# KL02: 三层分离 + 增长驱动的组织策略是架构设计的两条主轴

> 所属视角：P01 - 个人知识库的结构化方法
> 上层论点：塔尖

## 支撑论点

### 2.1: 输入→处理→输出三层分离，每层职责单一

- 逻辑顺序：结构（从宏观架构到具体层）
- 引用 atoms: KA-06, KA-07, KA-08
- 引用 groups: G03

原始素材（journal）只管记录不做修改（KA-06）；结构化拆解（pyramid）只管处理；面向读者的产出（outputs）只管表达（KA-07）。教程等产出不应与拆解过程同级，而是作为某个视角的输出（KA-08）。三层分离的核心价值是职责清晰、互不干扰。

### 2.2: 分析共享 + 表达隔离——pyramid 内部的双轨设计

- 逻辑顺序：结构（从共享到隔离）
- 引用 atoms: KA-09, KA-10, KA-12
- 引用 groups: G04

pyramid 内部进一步分为 analysis（自下而上，所有视角共享的素材池）和 structure（自上而下，各视角独立的组织方案）（KA-09）。关键洞察是 synthesis 的终点不等于 SCQA 的起点——"发现了什么"和"要传达什么"是两种不同视角（KA-10）。因此 structure 按视角分目录而非按产物类型分，支持不同读者看到不同金字塔（KA-12）。

### 2.3: 增长驱动的组织策略——会膨胀的用目录，固定的用单文件

- 逻辑顺序：程度（从通用原则到具体案例）
- 引用 atoms: KA-11, KA-13, KA-14, KA-15
- 引用 groups: G05

判断准则是"产物是否随素材规模增长"（KA-14）。增长型产物如 atoms 按月分目录（KA-11）、tree 按 Key Line 分文件（KA-13）；固定型产物如 SCQA 保持单文件，因为四要素构成紧密逻辑链，拆开会破坏整体验证价值（KA-15）。

## 论点间关系

2.1→2.2→2.3 构成结构顺序：先建立宏观三层架构（2.1），再深入 pyramid 内部的双轨设计（2.2），最后解决"具体到每个产物怎么存放"的微观策略（2.3）。粒度从粗到细递进。
