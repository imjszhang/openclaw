# js-eyes install.sh 安装脚本修复说明

> 来源：[../../../../journal/2026-02-28/js-eyes-install-script-fix.md](../../../../journal/2026-02-28/js-eyes-install-script-fix.md)
> 缩写：JS

## Atoms

| 编号  | 类型 | 内容                                                                                                   | 原文定位                          |
| ----- | ---- | ------------------------------------------------------------------------------------------------------ | --------------------------------- | ------------------------------------------------ | ------------------------ |
| JS-01 | 事实 | OpenClaw 的安全检查机制会拒绝加载任何路径或文件具有 world-writable (mode & 0o002) 属性的插件           | 根因分析 > 权限链路               |
| JS-02 | 事实 | zip 包内文件若权限为 0666，使用 unzip 或 python3 zipfile.extractall() 解压后会保留该不安全权限         | 根因分析 > 权限链路               |
| JS-03 | 判断 | 配置校验报错"plugin not found"的根本原因是插件因权限不安全被拦截，导致未注册到 knownIds 列表中         | 根因分析 > 权限链路               |
| JS-04 | 经验 | js-eyes 主技能通常通过 ClawHub 安装且权限正常，而子技能因 install.sh 解压 zip 包易触发权限问题         | 根因分析 > 主技能 vs 子技能       |
| JS-05 | 步骤 | 必须在 install.sh 每次解压完成后立即执行 find 命令，将文件权限设为 644，目录权限设为 755               | 必须修复：解压后权限收紧          |
| JS-06 | 步骤 | 权限收紧命令需添加 `2>/dev/null                                                                        |                                   | true` 后缀，防止因极端无权限情况导致安装脚本中断 | 必须修复：解压后权限收紧 |
| JS-07 | 经验 | 权限收紧操作建议放在解压完成之后、npm install 之前，避免后续步骤再次修改文件权限                       | 必须修复：解压后权限收紧          |
| JS-08 | 判断 | 当前 install.sh 使用相对路径 `./skills` 作为默认安装目录，依赖执行时的 CWD，易导致安装位置错误         | 可选改进 > 默认安装路径           |
| JS-09 | 经验 | 手动配置 openclaw.json 易出错，建议在脚本中增加自动检测并合并写入 plugins.load.paths 和 entries 的逻辑 | 可选改进 > 自动写入 openclaw.json |
| JS-10 | 步骤 | 已安装环境若遇插件加载失败，可手动对插件目录执行 chmod 644/755 修复权限后重启 daemon                   | 已安装环境的快速修复              |
