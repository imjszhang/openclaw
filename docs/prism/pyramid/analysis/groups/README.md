# 归组（Groups）

跨文档边界的信息单元分组。每个 group 将来自不同 journal 的 atoms 按内在逻辑归为一组，并提炼一句观点句。

## 命名规则

- 文件名格式：`G01-主题词.md`
- 编号（G01、G02...）仅做标识，不代表阅读顺序或重要性排序
- 主题词用简短的中文或英文，概括该组的核心主题

→ [分组总览与变更日志](INDEX.md)

## Group 文件模板

每个 group 文件遵循以下结构：

```markdown
# G01: [观点句]

> 一句有态度的判断句，概括这组信息单元说明了什么。

## 包含的 Atoms

| 编号  | 来源                     | 内容摘要 |
| ----- | ------------------------ | -------- |
| AR-01 | openclaw-analysis-report | ...      |
| FM-03 | fork-management-guide    | ...      |

## 组内逻辑顺序

说明这些 atoms 为什么按这个顺序排列（时间顺序 / 结构顺序 / 程度顺序）。
```
