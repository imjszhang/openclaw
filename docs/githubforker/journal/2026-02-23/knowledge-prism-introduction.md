# JS's Knowledge Prism（知识棱镜）

> 一套基于金字塔原理的三层知识蒸馏体系，将时间线笔记转化为结构化知识产出。

## 为什么叫"棱镜"

一束白光射入三棱镜，折射出有序的七色光谱。

这个物理过程恰好是这套系统的隐喻——散乱的日常笔记（白光）经过结构化拆解（棱镜），产出面向读者的清晰文章（光谱）。三棱镜的三角形截面，也暗合了金字塔原理的核心形态。

## 它解决什么问题

持续记笔记的人迟早会遇到同一个困境：**笔记越多，越难用**。

按日期记录的笔记天然是时间线结构——适合回忆"那天学了什么"，但不适合回答"关于 X 话题我知道些什么"。当笔记积累到几十篇以上，知识碎片散落在不同日期的不同文档里，检索靠关键词碰运气，复用靠记忆力硬扛。

知识棱镜的回答是：**别在原始笔记上硬加结构，而是建一条从素材到产出的流水线**。

## 三层架构一览

```
journal/   原始素材层    ← 按日期忠实记录，只增不改
    ↓
pyramid/   结构化拆解层  ← 基于金字塔原理，双轨处理
    ↓
outputs/   读者产出层    ← 面向特定读者的成品文章
```

### 第一层：journal（输入）

按日期归档的学习笔记，保留完整的探索过程。一个日期目录下可以有多篇文档，写完就归档，不回头修改。这一层的价值是**忠实**——它是所有后续分析的原始证据。

### 第二层：pyramid（处理）

这是整个体系的核心引擎，分为两条轨道：

**自下而上（analysis）——从碎片到观点：**

| 阶段      | 做什么                                      | 产出                 |
| --------- | ------------------------------------------- | -------------------- |
| Atoms     | 把 journal 中的每个知识点拆成独立的信息单元 | 按月归档的原子卡片   |
| Groups    | 跨文档边界归组，每组提炼一个观点句          | G01、G02... 归组文件 |
| Synthesis | 从观点句中收敛出顶层观点候选                | 候选列表 + 关系图    |

**自上而下（structure）——从观点到文章骨架：**

| 阶段       | 做什么                                      | 产出                    |
| ---------- | ------------------------------------------- | ----------------------- |
| SCQA       | 设计序言（情境-冲突-问题-答案）             | 一份读者画像 + 叙事入口 |
| Tree       | 展开金字塔全树（塔尖 → Key Lines → 支撑点） | 文章的完整逻辑骨架      |
| Validation | MECE 检查 + 逻辑链验证 + 缺口识别           | 质量报告                |

两条轨道的关系：analysis 是所有视角共享的公共素材池，structure 按视角独立——同一批分析结果可以支撑不同角度的输出（教程、博客、演讲稿等）。

### 第三层：outputs（输出）

基于 pyramid 视角产出的成品。每个产出目录对应一个视角，内部按章节编号。读者只需要看这一层，不需要知道背后的拆解过程。

## 增量演化：核心工作流

每次新增一篇 journal 笔记，触发一轮增量拆解：

```
写一篇 journal
    → 提取 atoms（有新知识点吗？）
    → 审视 groups（现有分组需要更新吗？需要新建分组吗？）
    → 检查 synthesis（顶层观点需要调整吗？）
    → 检查各视角（文章骨架需要补充吗？）
    → 追加修订记录
```

这套流程的设计理念是**持续蒸馏**——不追求一步到位，而是每次新输入都经过完整的处理管线，让知识体系在使用中自然生长。

## 三个设计原则

### 1. 关注点分离

三层之间职责清晰：journal 只管记录，pyramid 只管结构，outputs 只管表达。改动一层不影响其他层。

### 2. 增长驱动的组织策略

会增长的内容用目录拆分（atoms 按月建子目录，tree 按 Key Line 拆文件），固定的内容用单文件保内聚（SCQA、validation 保持单文件）。目录结构服务于内容规模，而非预设分类。

### 3. 可追溯性

每个层级的文件都带修订记录，atoms 标注来源 journal，groups 标注包含的 atoms，synthesis 标注支撑的 groups。任何一个输出结论都可以沿链路追溯到原始笔记。

## 适合谁用

- **有持续记笔记习惯的学习者**：笔记积累到一定量后想"做点什么"，但不知道从何下手。
- **想把个人知识整理成文章/教程的写作者**：有素材但缺乏从素材到成品的系统方法。
- **对金字塔原理感兴趣但不知道怎么落地的实践者**：这套体系是金字塔原理在个人知识管理中的一种具体实现。

不需要特定工具。纯 Markdown 文件 + 任意文件管理方式（Git、本地文件夹、Obsidian 等）即可运行。

## 实际产出示例

基于这套体系，已经完成的第一个产出是**《个人知识库的结构化方法》**——一篇四章的方法论文章，从 21 篇 journal 笔记中蒸馏而来：

| 章节   | 主题                         | 对应的问题                   |
| ------ | ---------------------------- | ---------------------------- |
| 引言   | 从时间线笔记到结构化知识体系 | 你可能遇到了什么问题？       |
| 第一章 | 为什么必须结构化重组         | 时间线组织差在哪？           |
| 第二章 | 架构设计的两条主轴           | 三层怎么分？文件怎么组织？   |
| 第三章 | 保障体系持续运转             | 怎么做到每次新增都有章可循？ |

从原始笔记到最终文章，经历了 22 个 atoms 提取、11 个 groups 归组、6 个 synthesis 候选、1 个完整视角的 SCQA + 金字塔树 + 验证，全程可追溯。

## 一句话总结

**JS's Knowledge Prism 是一套将"我今天学了什么"转化为"关于这个话题，体系化的认知是什么"的工作方法。**

它不改变你记笔记的方式，而是在笔记之上架设一条从碎片到结构、从结构到产出的蒸馏管线。
