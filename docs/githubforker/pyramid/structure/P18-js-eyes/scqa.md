| 核心诉求 | 寻求一种既能复用用户日常浏览器环境（保留登录态），又能实现安全可控远程控制的解决方案 |

## S - 情境（Situation）

在构建 AI Agent 自动化流程时，内置的沙箱浏览器实例虽然提供了良好的隔离性，但无法直接复用用户日常浏览器中已建立的登录态（如 Cookie、Session），导致在处理需要身份验证的网页交互时效率低下且体验割裂。

## C - 冲突（Complication）

若为每个任务启动独立的浏览器实例，不仅资源开销巨大，更无法解决“带登录态交互”的核心痛点；而直接操作用户主浏览器又面临安全风险与状态混乱的隐患，现有架构缺乏一种兼顾“环境复用”与“安全控制”的中间层机制。

## Q - 疑问（Question）

如何填补内置沙箱浏览器在“带登录态交互”场景的能力空白，实现用户浏览器环境的安全复用与协同控制？

## A - 答案（Answer）

JS-Eyes 通过 WebSocket 复用用户浏览器环境，填补了内置沙箱浏览器在“带登录态交互”场景的能力空白，支持多浏览器协同与远程控制。
