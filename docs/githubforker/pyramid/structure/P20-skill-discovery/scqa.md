## S - 情境（Situation）

在 OpenClaw 生态中，技能（Skills）是扩展 Agent 能力的核心单元，其有效性依赖于被 Agent 及时、准确地发现与加载。当前系统已具备标准化的技能元数据声明与语义搜索索引能力，为技能的流通奠定了数据基础。

## C - 冲突（Complication）

然而，现有的技能发现机制仍停留在“手动注册”阶段，依赖人工更新注册表，导致新技能上线滞后、元数据易出错，且无法适应 Agent 高频、实时的工具检索需求，严重制约了生态的自动化扩展能力。

## Q - 疑问（Question）

如何重构技能发现系统，使其从人工维护演进为自动化架构，以同时满足 Agent 的高效调用与人类开发者的可读性需求？

## A - 答案（Answer）

技能发现系统应从“手动注册”演进为"GitHub Pages 静态注册表 + AI 工具链”的自动化架构，实现 Agent 友好与人类可读的双重服务。
