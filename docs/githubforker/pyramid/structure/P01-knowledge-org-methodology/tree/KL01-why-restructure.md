# KL01: 时间线组织只服务于"回忆过程"，无法支撑"传递知识"——必须结构化重组

> 所属视角：P01 - 个人知识库的结构化方法
> 上层论点：塔尖

## 支撑论点

### 1.1: 按日期排列的笔记随规模增长暴露四个结构性问题

- 逻辑顺序：程度（从现象到诊断）
- 引用 atoms: KA-01, KA-02
- 引用 groups: G01

读者先感受到"笔记多了但不好用"（KA-01），再被诊断出四个具体问题：无阅读顺序、主题散落、深浅混杂、缺少中心论点（KA-02）。这四个问题共同指向一个根因：时间线是记录维度而非知识维度。

### 1.2: 金字塔原理提供了从素材到结构的系统化方法

- 逻辑顺序：结构（方法论组成）
- 引用 atoms: KA-03, KA-04, KA-05
- 引用 groups: G02

自上而下适合结论清晰时组织表达（KA-03），自下而上适合结论不确定时从素材中发现规律（KA-04）。实践中两者交替使用——先归纳再验证——形成完整的结构化循环（KA-05）。

## 论点间关系

1.1 和 1.2 构成"问题→工具"的结构顺序：1.1 诊断出问题，1.2 引入解决问题的方法论工具。读者看完 1.1 会追问"那怎么办"，1.2 回答"用金字塔原理的双向方法"。
